# 非编程者用AI开发项目工作流

## 33 步展开清单

### 一、方向与流程

| 文件名        | 内容                         |
| ---------- | -------------------------- |
| `产品定位.md`  | 目标用户、核心问题、最小价值、最小闭环、暂不做什么  |
| `主业务流程.md` | 主流程、支线、输入、处理、输出、反馈，以及当前主流程 |

1. 确定产品定位，把最小价值、最小闭环、暂不做什么说清楚。
2. 把业务流程讲清楚，包括主流程、支线、输入、处理、输出、反馈。
3. 从业务流程中抽出当前主流程，明确起点、关键动作、系统处理和成功结果。

### 二、页面骨架与 low-fi

| 步骤  | 文件名         | 内容                            |
| --- | ----------- | ----------------------------- |
| 1   | `页面清单.md`   | 全部必要页面，以及每个页面为什么存在            |
| 2   | `页面清单.md`   | 每个页面的主目标、核心区块、关键信息、关键操作       |
| 3   | 第一轮 low-fi  | 只验证页面结构、区块层级和主操作位置；组件先占位，不补细节 |
| 4   | `页面流转.md`   | 页面之间的跳转关系、状态流转、主流程经过哪些页面      |
| 5   | 做第二轮 low-fi | 在结构不变的前提下补齐核心组件表达和关键状态        |

4. 定页面清单，先定页面是否存在，再谈页面内容。
5. 逐页定义页面主目标，先定唯一职责。
6. 逐页定义页面核心区块，先定必要区块。
7. 逐区块定义关键信息与操作元素，先分清展示与行动。
8. 定义页面之间的跳转关系与状态流转。
9. 做第一轮 low-fi，只验证页面结构、区块层级和主操作位置；组件先占位，不补细节。
10. 做第二轮 low-fi，在结构不变的前提下补齐核心组件表达和关键状态。
11. 基于第二轮 low-fi 做评审和删减，删多余页面、区块、操作，补遗漏状态。

### 三、数据、设计系统与中间稿

12. 补后端数据模型文档，从页面与流程反推核心数据对象、字段、状态和关系。
13. 生成全量 mock 数据，并再次制作 low-fi mock。
14. 基于带数据的 low-fi 再做一轮评审和收敛，检查信息密度和字段优先级。
15. 制定 style guide，明确颜色、字体、间距、组件风格、状态表现和文案语气。
16. 基于 style guide，补组件初版规格，先覆盖按钮、输入框、卡片、标签等核心组件。
17. 做技术选型，基于产品结构、数据模型、style guide 和组件规格反推技术栈。
18. 做半高保真 mock，把 style guide 和技术约束落到页面。
19. 补组件最终规格，明确变体、状态、边界、响应式和实现约束。
20. 统一回写文档，形成“设计—数据—技术”的单一基线版本。

### 四、工程落地与扩展验证

21. 生成项目脚手架与目录结构，明确命名规则与分层规则。
22. 在项目脚手架中落地工程层原型，把页面、组件、路由和 mock 数据真正搭进代码结构。
23. 只优先实现主流程闭环，用最小真实实现验证工程结构、组件体系、数据模型和技术选型。
24. 基于主流程闭环实现结果，统一更新基线文档。
25. 实现第二条流程，验证组件复用、数据模型、工程结构与技术选型的可扩展性。
26. 再次收束文档与目录，形成 v2 基线。

### 五、持续迭代

27. 逐步放开剩余支线与外围功能，控制进入节奏。
28. 建立持续迭代规则，明确需求进入、优先级、改单范围、文档同步和验收规则。
29. 建立测试与回归检查清单，覆盖主流程、已实现流程、边界状态和设计一致性。
30. 建立发布前检查与版本节奏，明确什么状态算可以发布。
31. 建立反馈收集与下一轮决策机制，把外部声音变成结构化输入。
32. 定义必须拒绝做的事项，保护产品边界。
33. 沉淀自己的 AI 协作模板库，并定期复盘工作流本身，持续优化 AI 开发方法论。
