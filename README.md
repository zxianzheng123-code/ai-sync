# AI Tool Library

## 仓库定位

这是一个**个人自定义 AI 工具库**。

定位只有一句话：

**面向个人使用，持续沉淀可复用的 AI 技能、规则和方法资产，工具类型不设限制。**

这个仓库不是单一项目代码仓，也不是单一工作流产品。

它的核心职责是：

- 存放可复用的技能
- 存放全局规则
- 存放少量仓库级方法文档

## 使用者画像

本仓库的唯一使用者：**33 岁、零编程基础的校服零售品牌老板**。

- 经营规模：8 家直营店 + 1 仓库 + 线上网店，总部 1 人办公
- 技术能力：没有编程基础，通过 AI 工具（Claude Code / Codex）调用本仓库的技能
- AI 用途：解决业务问题、构建业务系统、调研分析、知识沉淀
- 当前阶段：探索期——每天使用 3 小时以上，但仍在学习如何设计更复杂的工作流
- 决策风格：稳妥优先——想清楚再动手，方案要说清楚利弊
- 掌控感：不接受黑盒——AI 替他做的任何判断，都要先让他看见、确认了再执行
- 产出期望：不只要结果，希望理解背后的逻辑，逐步提升自己的判断力
- 沟通方式：大白话，不懂技术术语

## 各目录职责

| 对象 | 职责 | 不负责 |
| --- | --- | --- |
| `README.md` | 仓库定位、目录结构、维护规则 | 不写对外触发说明 |
| `正哥技能召唤书.md` | 对外入口：技能列表、触发方式、协议索引 | 不承担仓库维护说明 |
| `对外规则库/` | 对外规则库：存放可复用的通用规则，供其他项目调用 | 不放仓库内部维护规则 |

| `临时资料/` | 临时存放的参考资料 | 不作为正式规则或技能引用 |

## 技能目录维护原则

`技能/` 层补充规则：

1. 一项技能一个目录。
2. 尽量保持扁平，不做深层分类目录。
3. 技能内部只放和技能直接相关的内容。

## 技能组织原则

这个仓库后续不按“项目类型”统一收口，而按“可复用能力”持续增加。

当前分两个层次：

### 共享协议层（所有技能的底层规范）

五大协议定义了所有技能的通用行为规范，各技能引用协议而非重复内联：

- `角色协议` — 心智模式、信念系统、核心叙事、习惯回路、角色分层
- `信息-认知协议` — 信息可靠性、全面性、及时性、偏差觉察与纠偏
- `决策-计划协议` — 决策框架、验收标准、SubAgent 分派、进度文件管理
- `执行-产出协议` — 输入校验、目标管理、协作质量、反馈闭环与复盘
- `通用审查协议` — 互撕门禁、三签、Lead 单次采纳

### 技能层（按使用场景分组）

- **日常经营**：`研究分析师`（调研对比分析）、`PLAYBOOK编写师`（知识沉淀）
- **造工具**：`造工具总控`（唯一入口）→ 调度 `需求规划师` → `产品设计师` → `全栈编码师` → `部署上线师` ← `代码维护师`
- **造技能**：`Skill编译器`（造新技能）
- **万能兜底**：`通用任务引擎`——上面都不合适时用这个

后续新增技能时，优先遵守这两个判断：

1. 这是可复用能力，还是一次性项目资料？
2. 这是独立技能，还是现有技能的参考附件？

如果只是一次性项目资料，不进入 `技能/`。

## 维护规则

### 文档分工

- 对外触发说明 → `正哥技能召唤书.md`
- 仓库维护说明 → `README.md`
- 不要两边各写一份，内容只放一个地方
- 技能正文写在 `SKILL.md`，长说明放 `参考资料/`
- 文档写作规范见 `对外规则库/通用文档写作规则.md`

### 命名规则

- 目录名、文件名默认使用**中文**，直接表达用途
- 同一层级命名口径必须一致，不混用中英文
- 受外部标准限制的固定文件名可保留原名（如 `README.md`、`SKILL.md`）
- 编程关键字、第三方产品名按原始要求保留，不强行翻译

### 技能增删改

**新建技能**：
1. 确认是可复用能力（不是一次性项目资料）
2. 确认现有技能无法覆盖（不重叠）
3. SKILL.md 必须包含：YAML frontmatter、TL;DR、角色设定、Strict Rules、Workflow、Initialization
4. 在 `正哥技能召唤书.md` 注册条目（技能名 + 路径 + 暗号），暗号不得冲突
5. 在 `技能/CHANGELOG.md` 记录

**修改技能**：
- 微调（文案/格式）→ 直接改，记录审计
- 局部（新增步骤/规则）→ 确认不破坏现有流程，记录审计
- 结构性（重命名/改输入输出/合并拆分）→ 同步更新 `正哥技能召唤书.md` + README + 下游引用，记录审计

**删除技能**：
1. 确认无下游依赖
2. 从 `正哥技能召唤书.md` 和 README 移除
3. 删除目录，记录审计

**修改共享协议**：
1. 列出所有引用该协议的技能
2. 确认修改后不退化
3. 记录审计

### 安全约束

- 删除操作须向用户确认
- 暗号不得重复
- `正哥技能召唤书.md` 中路径必须指向实际存在的 SKILL.md
- `对外规则库/` 目录下的文件只能通过用户明确指令修改

### 审计日志

所有变更记录在 `技能/CHANGELOG.md`，格式：`[操作类型] 技能名` + 操作者 + 变更摘要 + 影响范围 + 门禁校验。操作类型：CREATE / UPDATE / DELETE / MERGE / SPLIT / PROTOCOL_UPDATE。

## 当前状态

仓库处于**探索期**：通用能力框架已就位，垂直业务技能（如线上运营、门店标准化）待摸索清楚后再沉淀。新增技能通过 `Skill编译器` 生成。
