# AI Tool Library

## 仓库定位

这是一个**个人自定义 AI 工具库**。

定位只有一句话：

**面向个人使用，持续沉淀可复用的 AI 技能、规则和方法资产，工具类型不设限制。**

这个仓库不是单一项目代码仓，也不是单一工作流产品。

它的核心职责是：

- 存放可复用的技能
- 存放全局规则
- 存放少量仓库级方法文档

## 使用者画像

本仓库的唯一使用者：**33 岁、零编程基础的校服零售品牌老板**。

- 经营规模：8 家直营店 + 1 仓库 + 线上网店，总部 1 人办公
- 技术能力：没有编程基础，通过 AI 工具（Claude Code / Codex）调用本仓库的技能
- AI 用途：解决业务问题、构建业务系统、调研分析、知识沉淀
- 当前阶段：探索期——每天使用 3 小时以上，但仍在学习如何设计更复杂的工作流
- 决策风格：稳妥优先——想清楚再动手，方案要说清楚利弊
- 掌控感：不接受黑盒——AI 替他做的任何判断，都要先让他看见、确认了再执行
- 产出期望：不只要结果，希望理解背后的逻辑，逐步提升自己的判断力
- 沟通方式：大白话，不懂技术术语

## 文档分工

| 文档 | 面向对象 | 职责 |
| --- | --- | --- |
| `AGENTS.md` | 对外 | 可复制到其他项目的使用入口；说明有哪些技能、怎么触发、怎么读取 |
| `README.md` | 对内 | 说明本仓库怎么维护、目录怎么组织、哪些内容该放哪里 |

## 内部目录结构

根目录表达的是**本仓库内部维护结构**：

```text
ai-sync/
├── README.md                         # 仓库内部维护说明
├── AGENTS.md                         # 对外复制使用的入口文件
├── 规则/                            # 本仓库内部规则
│   └── *.md
├── 技能/                           # 全部可复用技能，保持扁平
│   └── <技能目录名>/
│       ├── SKILL.md
│       ├── 参考资料/
│       ├── 脚本/
│       └── 资源/
└── <仓库级文档>.md                   # 少量仓库级专题文档或讨论文档
```

## 内部维护边界

| 对象 | 这里负责什么 | 这里不负责什么 |
| --- | --- | --- |
| `README.md` | 说明仓库定位、内部目录、维护原则、内容归位 | 不写对外触发说明，不替代 `AGENTS.md` |
| `AGENTS.md` | 作为对外入口，给外部项目说明技能入口和触发方式 | 不承担仓库内部维护说明 |
| `规则/` | 沉淀本仓库构建与维护时使用的规则 | 不直接承担对外技能说明 |
| `技能/` | 存放全部可复用技能及其附属材料 | 不混入一次性项目资料 |

## 技能目录维护原则

`技能/` 层补充规则：

1. 一项技能一个目录。
2. 尽量保持扁平，不做深层分类目录。
3. 技能内部只放和技能直接相关的内容。

## 技能组织原则

这个仓库后续不按“项目类型”统一收口，而按“可复用能力”持续增加。

当前按使用场景分三组：

- **日常经营**：`研究分析师`（调研对比分析）、`PLAYBOOK编写师`（知识沉淀）
- **造工具**：`造工具总控`（唯一入口）→ 调度 `需求规划师` → `产品设计师` → `全栈编码师` → `部署上线师` ← `代码维护师`
- **造技能**：`Skill编译器`（造新技能）、`通用审查协议`（共享协议）
- **万能兜底**：`通用任务引擎`——上面都不合适时用这个

后续新增技能时，优先遵守这两个判断：

1. 这是可复用能力，还是一次性项目资料？
2. 这是独立技能，还是现有技能的参考附件？

如果只是一次性项目资料，不进入 `技能/`。

## 对内维护原则

- 对外怎么触发、怎么读取、有哪些技能，以 `AGENTS.md` 为准。
- 对内怎么组织目录、怎么写文档、哪些内容该放哪里，以 `README.md` 和 `规则/` 为准。
- 新增仓库级说明时，先判断它是“对外入口说明”还是“对内维护说明”，不要两边各写一份。
- 如果一条内容既能写进 `AGENTS.md`，也能写进 `README.md`，默认：
- 面向使用者的写进 `AGENTS.md`
  - 面向维护者的写进 `README.md`
- 技能正文写在 `SKILL.md`，长说明放进 `参考资料/`。
- 本项目构建与维护时的写作规则，以 `规则/通用文档写作规则.md` 为准。
