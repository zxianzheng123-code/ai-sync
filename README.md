# AI Tool Library

## 仓库定位

这是一个**个人自定义 AI 工具库**。

定位只有一句话：

**面向个人使用，持续沉淀可复用的 AI 工作流程、人设和方法资产，工具类型不设限制。**

这个仓库不是单一项目代码仓，也不是单一工作流产品。

它的核心职责是：

- 存放可复用的工作流程（技能）
- 存放可复用的工作人员人设（人设库）
- 存放 MCP 接入资产和 CLI 工具资产
- 存放护栏钩子和运维脚本
- 存放客户反馈报告

## 使用者画像

本仓库的唯一使用者：**33 岁、零编程基础的校服零售品牌老板**。

- 经营规模：8 家直营店 + 1 仓库 + 线上网店，总部 1 人办公
- 技术能力：没有编程基础，通过 AI 工具（Claude Code / Codex）调用本仓库的技能
- AI 用途：解决业务问题、构建业务系统、调研分析、知识沉淀
- 当前阶段：探索期——每天使用 3 小时以上，但仍在学习如何设计更复杂的工作流
- 决策风格：稳妥优先——想清楚再动手，方案要说清楚利弊
- 掌控感：不接受黑盒
- 沟通底线见 `协议/角色协议.md`

## 技能目录维护原则

`技能/` 层补充规则：

1. 一项技能一个目录。
2. 每个技能目录标准结构：`SKILL.md` + `作业指引/` + `资源/` + `脚本/`（含 `加载上下文.sh` 和 `验证/`）
3. 尽量保持扁平，不做深层分类目录。
4. 技能内部只放和技能直接相关的内容。

## 技能组织原则

### 人设库（工作人员）

集中存放在 `技能/人设库/`，各工作流程按需引用。人设只定义通用能力基线，流程差异写在 SKILL.md 步骤里。

### 技能层（按使用场景分组）

- **用户入口**：`客服工作`（唯一入口）→ 路由到下游
- **转化能力**：`问题拆解`（题目未成形时先拆）
- **答案与判断**：`答案澄清` / `调研判断`
- **沉淀能力**：`方法提炼` / `教材编写`
- **自动化能力**：`循环定时任务`（入口）→ `循环任务` / `定时任务` / `组合任务`
- **造工具**：`造工具`（入口）→ `需求规划` → `产品逻辑` → `产品设计` → `全栈编码` → `部署上线` ← `代码维护`
- **造技能**：`Skill编译` / `全景图生成`
- **质量保障**：`测试循环`
- **万能兜底**：`通用任务`

后续新增技能时，优先遵守两个判断：

1. 这是可复用能力，还是一次性项目资料？
2. 这是独立技能，还是现有技能的参考附件？

## 维护规则

### 文档分工

- 对外触发说明 → `正哥技能召唤书.md`
- MCP 工具接入说明 → `MCP/MCP接入总表.md`
- 仓库维护说明 → `README.md`
- 不要两边各写一份，内容只放一个地方
- 文档写作规范见 `对外规则库/通用文档写作规则.md`

### 命名规则

- 目录名、文件名默认使用**中文**，直接表达用途
- 同一层级命名口径必须一致，不混用中英文
- 受外部标准限制的固定文件名可保留原名（如 `README.md`、`SKILL.md`）
- `工具注册表.md`、`项目工具清单/`、`MCP/` 内的 `tool_id`、`project_id`、模板文件名统一使用 ASCII `kebab-case`

### 技能增删改

**新建技能**：
1. 确认是可复用能力（不是一次性项目资料）
2. 确认现有技能无法覆盖（不重叠）
3. SKILL.md 必须遵循 `AGENTS.md` 中的格式规范
4. 必须配 `脚本/加载上下文.sh`
5. 在 `正哥技能召唤书.md` 注册条目（技能名 + 路径 + 暗号），暗号不得冲突
6. 在 `技能/CHANGELOG.md` 记录

**修改技能**：
- 微调（文案/格式）→ 直接改，记录审计
- 结构性（重命名/改输入输出/合并拆分）→ 同步更新 `正哥技能召唤书.md` + README + 下游引用

**删除技能**：
1. 确认无下游依赖
2. 从 `正哥技能召唤书.md` 和 README 移除
3. 删除目录，记录审计

### 安全约束

- 删除操作须向用户确认
- 暗号不得重复
- `正哥技能召唤书.md` 中路径必须指向实际存在的 SKILL.md
- `对外规则库/` 目录下的文件只能通过用户明确指令修改

### 审计日志

技能变更记录在 `技能/CHANGELOG.md`，格式：`[操作类型] 技能名` + 操作者 + 变更摘要 + 影响范围。操作类型：CREATE / UPDATE / DELETE / MERGE / SPLIT / RESTRUCTURE。
