# 通用审查协议（已迁移）

> 本文件定义多个技能共用的审查规则与交叉检查循环模板。各 SKILL.md 引用本协议，不重复写入。

## 通用审查规则

1. **最低检查轮数**：重要内容至少检查 5 轮，其余至少 3 轮；未到轮数不得通过。为什么要这么多轮？因为检查越仔细，交到你手上的东西越靠谱。"重要内容"由技能的 SKILL.md 指定；未指定时默认 3 轮。
2. **全员确认**：检查阶段每轮都必须全部审查角色明确写出"通过"，否则不得进入下一步。
3. **独立审查→同时提交**：每位审查角色**先独立出结论**，不互相串通。所有角色**同时提交**各自的通过/不通过意见。主笔看到所有结论后统一处理。
4. **主笔单次采纳**：主笔每轮只采纳 1 条审查建议进行修订。多个审查角色同时提了不同问题时，主笔从中选 1 个最关键的修改，其他问题排队到下一轮（不丢弃）。
5. **指名道姓**：审查意见必须指出具体页面/组件/元素/文件名称，禁止笼统评价。
6. **必须找出问题**：每回合每位审查角色至少指出 1 个具体问题。若确实无问题，必须列出完整检查项清单（逐条说明检查了什么、结论是什么），证明不是没查而是真没问题。
7. **引用证据**：通过/不通过必须附带可验证证据。
8. **大白话沟通**：面向你的一切沟通禁止技术黑话。翻译官负责翻译。

## 分歧裁决

审查角色之间意见相反时，按以下流程处理：

1. 主笔选择采纳哪一方的建议（仍遵守单次采纳），并向被拒方说明理由
2. 被拒方有权坚持自己的意见
3. 坚持 2 轮后仍无法达成一致 → 主笔上报你裁决

上报格式：

```
[角色A] 认为：[具体意见]
[角色B] 认为：[具体意见]
我建议采纳 [谁的]，因为 [理由]。你来定。
```

## 交叉检查循环模板

所有检查阶段统一遵循以下流程，各技能指定轮数和确认角色即可：

```
第 1 轮（必做）：主笔提交初稿/草案 → 各专家指出 ≥1 个具体问题（指名道姓 + 引用证据）
第 2..N-1 轮（必做）：主笔采纳 1 条修订 → 各专家指出 ≥1 个具体问题
第 N 轮（必做）：主笔采纳 1 条修订
终审轮：各专家终审（指名道姓 + 引用证据）：
  - 仍有问题 → 回到第 N 轮
  - 全员确认通过（附检查清单），结束循环

每轮输出确认格式：
  [专家角色名 确认] 通过/不通过：<引用具体位置 + 理由>
```
