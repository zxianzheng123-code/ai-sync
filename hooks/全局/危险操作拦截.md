# 危险操作拦截

## 类型

全局（始终激活）

## 触发时机

在任何 Bash 命令执行**之前**触发，对即将运行的命令内容进行关键词匹配。

## 拦截规则

检测以下危险模式和关键词：

### 文件系统破坏
- `rm -rf /`、`rm -rf *`、`rm -rf .`
- 任何 `rm -rf` 后接根目录或通配符的组合
- `mkfs`、`dd if=`（覆盖磁盘）

### 数据库破坏
- `DROP TABLE`、`DROP DATABASE`
- `DELETE FROM`（不带 WHERE 条件）
- `TRUNCATE`、`TRUNCATE TABLE`

### Git 强制推送
- `git push --force`、`git push -f`
- `git push --force-with-lease`（仍需确认）
- `git reset --hard` 后接远程分支

### 其他高危操作
- `chmod -R 777`
- `chown -R` 应用于系统目录
- `kill -9 1`、`killall`
- `:(){ :|:& };:`（fork 炸弹）
- `> /dev/sda`

### 匹配方式

对命令字符串进行**不区分大小写**的子串/正则匹配。管道、别名、变量展开后的结果均需检测。

## 处理方式

1. **立即阻断**：不执行该命令。
2. **醒目警告**：向用户输出被拦截的命令内容及匹配到的危险模式。
3. **要求确认**：用户必须明确回复"确认执行"或等效指令后，才可放行。
4. **记录日志**：将拦截事件（时间、命令、匹配规则）写入会话日志，便于事后审计。

## 示例

以下命令会被拦截：

```bash
# 文件系统破坏
rm -rf /
rm -rf /home/user/important-project/*

# 数据库破坏
mysql -e "DROP TABLE users;"
psql -c "DELETE FROM orders;"
sqlite3 db.sqlite "TRUNCATE TABLE logs;"

# Git 强制推送
git push --force origin main
git push -f origin develop

# 其他
chmod -R 777 /etc
dd if=/dev/zero of=/dev/sda
```

以下命令**不会**被拦截（正常操作）：

```bash
rm -rf ./node_modules
rm temp.txt
git push origin feature/my-branch
DROP_COUNT=0   # 变量赋值，非 SQL 语句
```
