# 删除确认

## 类型

全局（始终激活）

## 触发时机

在任何涉及**文件或目录删除**的操作执行**之前**触发。包括但不限于：

- Bash 命令中的删除操作
- 代码编辑中通过脚本执行的删除
- Git 管理的文件删除

## 拦截规则

检测以下删除相关的命令和模式：

### Bash 命令
- `rm`（含所有参数组合：`-r`、`-f`、`-rf`、`-i` 等）
- `rmdir`
- `unlink`
- `shred`

### Git 操作
- `git rm`
- `git rm --cached`
- `git clean -fd`

### 代码级删除
- `os.remove()`、`os.unlink()`（Python）
- `fs.unlink()`、`fs.rmdir()`、`fs.rm()`（Node.js）
- `File.delete()`（Java/Kotlin）
- `std::fs::remove_file()`、`std::fs::remove_dir_all()`（Rust）

### 批量操作
- `find ... -delete`
- `find ... -exec rm`
- `xargs rm`

## 处理方式

1. **暂停执行**：命令不立即运行。
2. **列出目标**：向用户清楚展示即将被删除的文件或目录路径列表。
3. **请求确认**：等待用户明确回复同意后才执行。
4. **支持批量确认**：如果用户在同一轮对话中已确认过同类删除（如清理构建产物），可复用该确认，无需逐条重复询问。

## 示例

以下操作会触发确认：

```bash
# 单文件删除
rm config.yaml

# 递归删除目录
rm -rf build/

# Git 删除跟踪文件
git rm src/deprecated-module.ts

# 批量查找删除
find . -name "*.log" -delete

# 清理未跟踪文件
git clean -fd
```

确认对话示例：

```
[删除确认] 即将删除以下内容：
  - build/          （目录，包含 47 个文件）

请确认是否继续？(y/n)
```
