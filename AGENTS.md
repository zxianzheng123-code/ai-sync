# 技能触发指令

## 全局规则

每次任务开始前，先读取 `规则/` 目录下的所有 `.md` 文件，将其中的规则作为本次任务的**强制约束**。

- `规则/通用文档写作规则.md` — 写作规范
- `规则/技能管理规则.md` — 技能增删改权限与审计

## 共享协议（所有技能的底层规范）

所有技能都必须遵守以下五大协议。协议定义了技能的底层行为规范，各技能不再重复内联这些规则。

| 协议 | 管什么 | 路径 |
| --- | --- | --- |
| 角色协议 | 心智模式、信念系统（用户主权/透明不黑盒/不脑补）、核心叙事、习惯回路（提问/行动/汇报）、角色分层 | `技能/角色协议.md` |
| 信息-认知协议 | 信息来源分级、结构化摸底、盲区检查、时效标注、偏差觉察与纠偏 | `技能/信息-认知协议.md` |
| 决策-计划协议 | 信息充分性判断、多方案决策流程、验收标准编制、SubAgent分派、进度文件管理 | `技能/决策-计划协议.md` |
| 执行-产出协议 | 上游产物校验、目标管理、单步推进、自我校验、协作质量、反馈闭环与复盘 | `技能/执行-产出协议.md` |
| 通用审查协议 | 互撕门禁（≥5轮）、三签/多签、Lead单次采纳、指名道姓、引用证据 | `技能/通用审查协议.md` |

## 使用者上下文

本工具库的使用者是一个**没有编程基础的小企业老板**（校服零售品牌，8 直营店 + 1 仓库 + 线上网店）。所有面向客户的沟通默认大白话，禁止技术术语。

## 触发方式

有两种方式触发技能：

1. **说暗号**：直接说技能的固定暗号（如"帮我调研"），立刻进入该技能。
2. **说人话**：用自己的话描述想做什么（如"帮我看看竞品怎么做的"），通用任务引擎会判断该用哪个技能，**先告诉你"我打算用XX技能来做"，你确认了才执行**。
