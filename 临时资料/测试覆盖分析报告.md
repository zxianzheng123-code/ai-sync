# 测试覆盖分析报告

> 分析日期：2026-03-16

## 现状

项目目前只有一个验证脚本 `scripts/validate.sh`，包含 4 项检查，全部通过：

| # | 检查项 | 验证内容 |
|---|--------|----------|
| 1 | 技能路径有效性 | 召唤书中的路径指向真实的 SKILL.md |
| 2 | 暗号唯一性 | 无重复触发暗号 |
| 3 | CHANGELOG 时效性 | 30 天内有更新 |
| 4 | 目录完整性 | 每个技能目录都有 SKILL.md |

## 发现的 8 个验证盲区

### 1. 协议引用完整性（严重）

每个 SKILL.md 的 Strict Rules 应引用全部 6 个共享协议，但**所有 10 个技能都漏了工具链协议**。另外召唤书写的是"五大协议"，实际列了 6 个——文字也对不上。

### 2. SKILL.md 结构合规性

没有检查每个 SKILL.md 是否包含必要结构：

- YAML frontmatter（`name` + `description`）
- TL;DR（输入/输出/性质）
- Strict Rules 引用全部协议
- 末尾有 Initialization 段

### 3. 协议文件存在性

当前脚本只检查技能路径，从不检查协议文件是否存在（角色协议.md、信息-认知协议.md 等）。

### 4. 产出物注册表一致性

依赖图.md 列了 8 个产出物及其生产/消费关系，但没验证：

- SKILL.md 中提到的产出物是否都在注册表里
- 生产者/消费者技能名是否和实际目录一致

### 5. 召唤书 ↔ 技能目录双向检查

当前只做单向检查（召唤书 → SKILL.md 存在）。缺少：

- 反向检查：每个技能目录是否在召唤书中有对应条目
- 召唤书中的技能名是否和 SKILL.md 的 YAML `name` 字段一致

### 6. 角色协议子文件完整性

角色协议.md 引用了 3 个子协议（执行层/审查层/翻译层），没有检查这些文件是否存在。

### 7. 使用指引.md 引用有效性

使用指引.md 引用了 `对外规则库/` 下的文件，没有验证引用的文件是否真的存在。

### 8. Markdown 内部链接有效性

没有检查文档中的相对链接是否能正确解析到目标文件。

## 实际发现的 Bug

分析过程中发现了一个真实问题：

- **正哥技能召唤书.md** 写的是"五大协议"，但表格列了 **6 个**协议
- **全部 10 个 SKILL.md** 的 Strict Rules 都**漏引了工具链协议**
- 这对造工具总控（9 处 SubAgent 引用、18 处进度文件引用）和通用任务引擎（4 处 SubAgent 引用）影响最大

## 建议优先级

| 优先级 | 新增检查 | 原因 |
|--------|----------|------|
| P0 | 协议引用完整性 | 已发现实际 bug |
| P0 | 协议文件存在性 | 系统根基 |
| P1 | SKILL.md 结构合规 | 防止技能残缺 |
| P1 | 召唤书 ↔ 目录双向检查 | 防止孤立技能 |
| P2 | 产出物注册表一致性 | 防止数据流断裂 |
| P2 | 角色子协议存在性 | 防止角色体系断裂 |
| P3 | 使用指引引用有效性 | 防止接入文档失效 |
| P3 | Markdown 链接验证 | 整体质量 |
