# 文档架构

## 文档架构设计逻辑

文档按“项目根目录总纲、参考文档明细、项目治理文档”三层组织。  
项目根目录只放总入口与主线文档；详细说明统一收进 `参考文档/`；阶段性收束、验证、发布规则统一收进 `项目治理/`。

## 界面层级定义

| 层级 | 对应界面 |
| --- | --- |
| `应用层` | `业务模块页面` |
| `对象层` | `业务实体页面` |
| `功能操作层` | `对象功能操作界面` |

## 文档类型说明

| 文档类型         | 文件名             | 章节                                 | 深度和边界                          |
| ------------ | --------------- | ---------------------------------- | ------------------------------ |
| `项目入口`       | `README.md`     | `项目目标`、`当前范围`、`文档入口`、`核心文档入口`      | 只做项目总览与导航；不展开业务规则、页面细节、技术实现。   |
| `功能需求`       | `功能需求.md`       | `功能目标`、`功能范围`、`核心功能`、`不做项`         | 只写做什么与不做什么；不展开页面布局、字段规则、实现方案。  |
| `主业务流程`      | `主业务流程.md`      | `主流程`、`支线`、`输入`、`处理`、`输出`、`反馈`     | 只写流程主线与输入输出；不展开页面布局、组件结构、技术实现。 |
| `mock原件`     | `最终mock/`       | `最终视觉真源`、`最终交互真源`                  | 只保留最终视觉与交互结果；不承接正文规则解释。        |
| `页面布局`       | `页面布局.md`       | `所属层级`、`页面层级`、`区域划分`、`主操作区`、`布局边界` | 只写本层级界面布局与区块边界；不展开组件内部结构和交互细节。 |
| `布局说明`       | `布局说明.md`       | `布局口径`                             | 只写区域层级和布局原则；不展开组件交互、数据规则。      |
| `通用组件`       | `<组件名>.md`      | `结构说明`、`交互说明`、`动效说明`               | 写到组件级结构、状态、交互为止；不承接页面专属业务规则。   |
| `专用组件`       | `<组件名>.md`      | `结构说明`、`交互说明`、`动效说明`               | 只写当前场景专用组件；不抽象成全局通用规范。         |
| `数据库架构`      | `数据库架构.md`      | `对象`、`存储位置`、`是否派生`、`是否持久化`         | 只写对象层与存储关系；不展开字段业务解释和页面消费方式。   |
| `数据字典`       | `数据字典.md`       | `字段归属`、`字段含义`、`来源`、`去向`            | 写到字段定义与流向为止；不重复流程说明和页面规则。      |
| `状态字典`       | `状态字典.md`       | `状态对象`、`状态值`、`进入条件`、`退出条件`         | 只写状态定义与切换条件；不展开页面样式和实现代码。      |
| `分类数据说明`     | `<分类名>数据说明.md`  | `核心对象`、`源数据`、`派生数据及派生规则`           | 只展开某一类数据对象；不重复全局数据架构。          |
| `技术选型`       | `技术选型.md`       | `前端`、`后端`、`数据库`、`鉴权`、`AI调用`、`部署方案` | 只写技术决策与选型边界；不展开目录明细和模块实现。      |
| `项目文件架构`     | `项目文件架构.md`     | `前端目录`、`后端目录`、`本地资源`、`外部依赖`        | 只写目录职责与资源归属；不展开业务规则和字段定义。      |
| `功能模块架构`     | `功能模块架构.md`     | `模块逻辑图`、`功能模块一览表`                  | 只写模块拆分与依赖关系；不展开页面细节和字段级规则。     |
| `基线修订记录`     | `基线修订记录.md`     | `变更项`、`变更原因`、`影响范围`、`生效时间`         | 只记变更结论与影响；不重复完整正文。             |
| `测试与回归清单`    | `测试与回归清单.md`    | `主流程检查项`、`边界状态检查项`、`设计一致性检查项`      | 只列验证项与通过标准；不承接需求说明和实现设计。       |
| `发布前检查与版本节奏` | `发布前检查与版本节奏.md` | `发布检查`、`版本分级`、`发布节奏`、`可发布标准`       | 只写发布门槛与节奏规则；不展开功能正文。           |
