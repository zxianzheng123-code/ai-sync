# 中文化路径与命名规则

## 1. 文档语言规则

- 本项目新增和修订的业务文档、说明文档、规则文档，默认使用中文命名与中文正文。
- 文档中的目录名、文件名、章节名、表头名、说明语句，优先使用中文口径，不额外混入英文别名。
- 只有在外部协议、工具约束、技术关键字不可替代时，才保留最小必要的英文原词。

## 2. 路径命名规则

- 本项目内新增目录、文件路径，默认使用中文命名。
- 路径命名应直接表达对象或用途，不使用无语义缩写，不使用拼音过渡名，不为了“技术味”保留英文 slug。
- 同一层级的命名口径必须一致；既然该层使用中文，就不得同时混入同级英文目录名。

## 3. Skill 路径规则

- `技能/` 下的 Skill 目录默认使用中文命名，与用户看到的工具名称保持一致。
- Skill 的子目录、阶段目录、运维目录，只要属于本项目自定义结构，也应使用中文命名。
- 如果 Skill 目录、文档表述、引用路径三者不一致，必须以“统一改到中文”为目标修正，不允许长期并存两套口径。

## 4. 例外边界

- 受外部标准限制的固定文件名可保留原名，例如 `README.md`、`AGENTS.md`、`SKILL.md`。
- 编程语言关键字、命令行参数、第三方产品名、协议字段名，按原始要求保留，不强行翻译。
- 例外只针对不可替代的固定对象，不得把“习惯”当成例外理由。

## 5. 执行要求

- 新建内容时，先判断名称是否可直接中文化；可以中文化的，不先写英文再回头补改。
- 发现已有英文路径或英文目录与本规则冲突时，后续任务可按实际影响逐步回收，但新增内容不得继续扩散旧口径。
- 文档中的展示名称、仓库中的真实路径、规则中的正式叫法，应尽量保持同一套中文口径。
