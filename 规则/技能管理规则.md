# 技能管理规则

> 本规则定义技能模块的增删改操作权限、校验门禁和审计机制。所有对 `技能/` 目录的变更必须遵守本规则。

## 一、管辖范围

本规则管辖以下对象的一切变更：

| 对象 | 路径 | 变更类型 |
|------|------|----------|
| 技能 SKILL.md | `技能/<技能名>/SKILL.md` | 新建 / 修改 / 删除 |
| 技能附属文件 | `技能/<技能名>/参考资料/*` | 新建 / 修改 / 删除 |
| 共享协议 | `技能/通用审查协议.md` 等 | 修改 / 删除 |
| 技能注册表 | `AGENTS.md` 技能列表 | 新增条目 / 删除条目 / 修改条目 |

## 二、操作权限与门禁

### 2.1 新建技能（CREATE）

**权限**：开放。满足以下门禁即可创建。

**前置门禁**：
1. **独立性判断**：确认是可复用能力而非一次性项目资料。
2. **去重判断**：确认现有技能无法覆盖该能力，不与已有技能功能重叠。
3. **命名规范**：目录名使用中文，表达技能用途。

**结构门禁**（SKILL.md 必须包含）：
1. YAML frontmatter（`name` + `description`）
2. TL;DR（输入/输出/性质/关键卖点）
3. 角色设定
4. Strict Rules（自包含或引用共享协议）
5. Workflow（Step 编号连续）
6. Initialization

**注册门禁**：
1. 在 `AGENTS.md` 技能列表添加条目（技能名 + 路径 + 固定暗号）。
2. 固定暗号不得与已有暗号冲突。
3. 在 `README.md` 能力分类中归入对应类别。

**审计**：在 `技能/CHANGELOG.md` 记录 CREATE 条目。

---

### 2.2 修改技能（UPDATE）

**权限**：开放。满足以下门禁即可修改。

**前置门禁**：
1. **功能完整性保障**：修改不得删除现有功能点，除非明确替换为等效或更优方案。
2. **接口兼容性**：若修改了输入/输出文件名或格式，必须同步更新所有引用该技能产出的下游技能。
3. **共享协议同步**：若技能引用了共享协议（如 `通用审查协议.md`），修改不得与协议冲突。

**修改分级**：

| 级别 | 定义 | 门禁 |
|------|------|------|
| 微调 | 文案/格式/拼写修正，不影响工作流 | 直接修改，记录审计 |
| 局部 | 新增/调整 Strict Rules 或 Workflow 步骤 | 确认不破坏现有流程，记录审计 |
| 结构性 | 重命名技能/改变输入输出/合并拆分 | 须同步更新 AGENTS.md + README.md + 下游引用，记录审计 |

**审计**：在 `技能/CHANGELOG.md` 记录 UPDATE 条目。

---

### 2.3 删除技能（DELETE）

**权限**：开放。满足以下门禁即可删除。

**前置门禁**：
1. **依赖检查**：确认无其他技能的输入依赖该技能的输出。
2. **引用检查**：确认无共享协议或 AGENTS.md 以外的文件引用该技能。
3. **能力覆盖确认**：确认该技能的能力已被其他技能覆盖，或用户明确不再需要。

**执行步骤**：
1. 从 `AGENTS.md` 技能列表移除条目。
2. 从 `README.md` 能力分类移除。
3. 删除 `技能/<技能名>/` 整个目录。
4. 在 `技能/CHANGELOG.md` 记录 DELETE 条目。

---

### 2.4 修改共享协议（PROTOCOL UPDATE）

**权限**：开放。满足以下门禁即可修改。

**前置门禁**：
1. **影响面评估**：列出所有引用该协议的技能。
2. **兼容性确认**：修改后各引用技能的行为不得退化。
3. **若删除规则**：确认所有引用技能已内联或不再需要该规则。

**审计**：在 `技能/CHANGELOG.md` 记录 PROTOCOL_UPDATE 条目。

---

## 三、审计日志规范

所有变更记录在 `技能/CHANGELOG.md`，格式如下：

```markdown
## YYYY-MM-DD

### [操作类型] 技能名/协议名
- **操作者**：AI / 用户指令
- **变更摘要**：一句话描述
- **影响范围**：列出受影响的文件
- **门禁校验**：通过 / 豁免（附理由）
```

操作类型枚举：`CREATE` / `UPDATE` / `DELETE` / `MERGE` / `SPLIT` / `PROTOCOL_UPDATE`

---

## 四、安全约束

1. **不可逆操作确认**：删除技能前须向用户确认。批量删除须逐个确认。
2. **暗号唯一性**：`AGENTS.md` 中不得出现重复暗号。
3. **路径一致性**：`AGENTS.md` 中的路径必须指向实际存在的 SKILL.md。
4. **规则文件只读**：`规则/` 目录下的文件只能通过用户明确指令修改，AI 不得自行变更。
5. **共享协议保护**：删除共享协议前，须确认无技能引用。

## 五、定期巡检

建议定期执行以下检查：

1. **注册一致性**：AGENTS.md 条目与 `技能/` 目录 1:1 对应。
2. **引用完整性**：共享协议被引用次数 > 0（否则可考虑清理）。
3. **暗号冲突检测**：无重复暗号。
4. **孤儿目录检测**：`技能/` 下无未注册的技能目录。
5. **产出文件名一致性**：上游技能输出文件名 = 下游技能输入文件名。
