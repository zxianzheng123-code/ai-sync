# 使用指引

## 接入技能系统

在其他项目的 `AGENTS.md`（Codex）或 `CLAUDE.md`（Claude Code）末尾加上这段：

```markdown
## 技能系统

执行任务前，先读取 `/Users/zxz/Desktop/ai-sync/正哥技能召唤书.md` 并遵守其中的全部规则。
```

加完就能用了，不用把整个文件复制过去。

## 接入 MCP 系统

在其他项目的 `AGENTS.md`（Codex）或 `CLAUDE.md`（Claude Code）末尾再加一段：

```markdown
## MCP 系统

本项目的 `project_id` 是 `<project-id>`；执行涉及外部工具的任务前，先读取 `/Users/zxz/Desktop/ai-sync/MCP/MCP接入总表.md`，再按该 `project_id` 继续读取对应项目清单。
```

默认读取链是：

1. 项目的 `AGENTS.md` / `CLAUDE.md`
2. `/Users/zxz/Desktop/ai-sync/MCP/MCP接入总表.md`
3. `/Users/zxz/Desktop/ai-sync/MCP/项目清单/<project-id>.md`

如果这个项目没有专属清单，就回退到 `MCP/项目清单/default.md`。

## 准备 MCP 配置和密钥

中央密钥文件默认放在：

```text
~/.config/ai-sync/mcp-secrets.env
```

仓库里只维护模板：

- Claude Code 模板：`/Users/zxz/Desktop/ai-sync/MCP/配置模板/claude.mcp.json.example`
- Codex 模板：`/Users/zxz/Desktop/ai-sync/MCP/配置模板/codex.mcp.toml.example`
- 密钥模板：`/Users/zxz/Desktop/ai-sync/MCP/密钥模板/mcp-secrets.env.example`

粘贴位置：

- Claude Code：把需要的服务片段写进 `~/.claude/mcp.json`
- Codex：把需要的 MCP 段写进 `~/.codex/config.toml`

不要把真实密钥写进本仓库；只在 `~/.config/ai-sync/mcp-secrets.env` 里维护真实值。

## 同时用 Claude Code 和 Codex

不需要两个文件写同样的内容。挑一个当主文件，另一个写一句话指过去就行：

**方式 A**：CLAUDE.md 当主文件，AGENTS.md 写：

```markdown
所有规则见 CLAUDE.md，请先读取该文件。
```

**方式 B**：AGENTS.md 当主文件，CLAUDE.md 写：

```markdown
所有规则见 AGENTS.md，请先读取该文件。
```

两种都行，看你习惯用哪个工具多就让哪个当主文件。

## 对外规则库

`对外规则库/` 目录存放可复用的通用规则，任何项目都能调用。

使用方法：在项目的 `AGENTS.md` 或 `CLAUDE.md` 中加上需要的规则引用，比如：

```markdown
执行任务前，先读取 `/Users/zxz/Desktop/ai-sync/对外规则库/通用文档写作规则.md` 作为写作规范。
```

当前可用规则：

| 规则 | 用途 |
|------|------|
| `通用文档写作规则.md` | 文档的逻辑、层级、信息密度、内容编写规范 |

## 触发技能

接入后有两种用法：

1. **说暗号**：直接说技能的固定暗号（如"开始转化"），立刻进入该技能
2. **说人话**：用自己的话描述想做什么（如"帮我看看竞品怎么做的"），AI 会判断用哪个技能，先告诉你再执行

完整的技能列表和暗号见 `正哥技能召唤书.md`。
