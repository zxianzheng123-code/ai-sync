## 执行模式
Master Agent 负责调度和验收，不亲自干活。每个步骤启动 SubAgent 执行。

## 工作人员
- 产品（→ 技能/人设库/产品/人设.md）
- 写作（→ 技能/人设库/写作/人设.md）

## 核心理念
- 蓝图是起跑线，原型才是终点线
- 不做美化：视觉风格、配色、字体等美术工作放到代码完成后单独做
- 原型 = 准生产级：去掉皮肤就是成品，不是"草图能跑"
- 原型一次性：原型代码是参考品，禁止复制进生产项目

## Workflow

### Step 0：加载上下文（Master Agent）
执行 脚本/加载上下文.sh，强制读取所有引用内容

### Step 1：设计规范
- 执行者：产品（SubAgent）
- 目标：分诊确认设计范围，读取上游产物，盘点页面并快速排版
- 成功标准：页面清单覆盖所有业务模块，每页有 CTA + 三段式骨架 + ASCII 草图
- 输入：`design/project_context.md` + `design/product_logic.md`
- 输出：`design/layout_blueprint.md`（布局蓝图，内部草稿）
- 执行逻辑：
  1. 分诊：向用户确认执行哪些阶段（A布局蓝图 / B可点击原型 / AB全流程）
  2. 读取业务文档，提取模块/动作/规则/场景/联动关系
  3. 盘点核心页面清单（必须覆盖 product_logic.md 中所有场景涉及的模块）
  4. 每页写 1 个 CTA + 三段式骨架（顶部固定/中间主体区滑动/底部固定）+ ASCII 草图
  5. 快速确认动态效果需求（对象/触发/目标/优先级）
  6. 向用户展示确认
- 关键原则：
  - 禁前端黑话：只用"最顶部/中间主体区/最底部"描述空间
  - 不新增业务内容：信息缺失只能提问补齐，不脑补
  - 蓝图是方向性草图，不用纠结细节

### Step 2：页面设计
- 执行者：产品（SubAgent）
- 目标：基于蓝图构建可点击的准生产级交互原型
- 成功标准：所有页面可点击体验，UI结构/间距/交互流程与最终产品一致（只差视觉皮肤）
- 输入：Step 1 布局蓝图 + 上游业务文档
- 输出：可点击原型（存入 `design/prototype/`）+ 交互地图
- 执行逻辑：
  1. 摄入蓝图，用产品经理视角审视合理性
  2. 造生产级 Mock 数据（字段完整、数据真实、量级对标、边界覆盖、跨页一致）
  3. 内部互撕产出交互地图
  4. 构建可点击原型（蓝图不合理的地方主动优化，记录偏离理由）
  5. 用户体验试玩：每页提供 3-5 个关键交互让用户测试
- Mock 数据质量标准：
  - 列表类数据至少 10-20 条
  - 每字段覆盖：正常值、长文本、短文本、空值、极端数值
  - 跨页引用数据保持一致

### Step 3：交互细节
- 执行者：写作（SubAgent）
- 目标：生成设计说明文档，记录截图无法表达的交互意图和逻辑
- 成功标准：每页有截图+补充说明，补充说明只写截图看不出来的东西
- 输入：Step 2 原型 + 交互地图
- 输出：`design/design_spec.md`（截图 + 设计说明）
- 补充说明维度：
  - 交互意图：为什么这样设计，用户预期的操作路径
  - 状态触发条件：什么条件下切换状态
  - 动效需求：哪些交互有动态效果
  - 跨页联动：操作影响哪个页面
  - 边界情况：截图没覆盖的极端场景

### Step 4：审查
- 执行者：产品（SubAgent）
- 目标：汇总技术发现，做交付校验
- 成功标准：发现清单已显式声明，所有校验项通过
- 输入：Step 2-3 所有产物
- 输出：发现清单（冲突项/扩展项/无影响项）
- 校验清单：
  - [ ] 所有页面有交互地图
  - [ ] 交互地图覆盖所有可交互元素
  - [ ] 发现清单已显式声明
  - [ ] Mock 数据对标生产
  - [ ] design_spec.md 每页有截图+补充说明

### Step 5：验证
- 如有 脚本/验证/ 下的脚本 → 自动执行
- 如有 作业指引/交付自查清单.md → 逐项检查
- 全部通过才进入交付

### Step 6：交付审核（Master Agent）
输出成品，等用户确认

交付物：
- 可点击原型（`design/prototype/`，一次性参考品）
- `design/design_spec.md`（截图 + 设计说明，下游编码主输入）
- `design/interaction_map.md`（交互地图）

交付口径：
> 交互体验版全部完成！所有页面验证通过。设计说明文档已生成，后续编码会参照截图和说明从干净骨架从零写代码。
