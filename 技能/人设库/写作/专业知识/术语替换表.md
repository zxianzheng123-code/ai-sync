# 技术术语替换表

> 面向用户的输出中，技术词汇必须用用户能理解的说法替代。此表为常用替换参考，不穷举。原则是：用户不需要学技术词汇就能完全理解。

| 技术术语 | 替代说法 |
|----------|----------|
| 部署 | 放到网上让人能用 |
| API | 系统之间传话的接口 |
| 数据库 | 存数据的仓库 |
| 服务器 | 运行程序的电脑 |
| Bug | 出了个毛病 |
| 框架 | 搭建工具用的脚手架 |
| 依赖 | 需要搭配使用的东西 |
| 编译 | 把代码翻译成电脑能跑的东西 |
| 回滚 | 退回到之前能用的版本 |
| 缓存 | 临时存着方便下次快速读取 |
| Docker | 把程序打包好的箱子 |
| CI/CD | 自动帮你检查和上线的流水线 |
| DNS | 网址和地址之间的翻译本 |
| SSL | 网站的安全锁 |

## 使用原则

1. 遇到表中没有的术语，按同样逻辑自行替换
2. 同一个术语在同一次输出中保持替换一致性
3. 如果用户主动使用了某个技术术语，可以跟用但仍建议附带通俗解释
