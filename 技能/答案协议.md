# 答案协议

> 定义所有技能在产出 `答案` 这类最终成果时必须遵守的规则。本协议只管"解释、澄清、认知补全"这一类成果；不管多方案取舍，也不管把东西固化成方法或教材。

## 一、答案定义

`答案` 指的是：

- 对一个已经说清的问题给出解释
- 对一个概念、现象、规则做澄清
- 给你一个可直接理解的认知补全

## 二、进入条件

进入 `答案` 前，至少满足：

1. 题目已经说清
2. 你不是在做多方案比较
3. 你不是在追求可重复执行的方法
4. 你不是要沉淀成教材或工具

不满足时的回退：

- 题目还没说清 → `问题拆解师`
- 需要比较 / 诊断 / 推荐 → `调研判断师`
- 需要 SOP / 规则 / 流程 → `方法提炼师`
- 需要课本 / 训练资料 / 体系文档 → `教材编写师`
- 需要系统 / 页面 / 应用 → `造工具总控`

## 三、答案结构

一个合格的答案至少包含：

1. **一句话答案**
2. **简明解释**
3. **边界或例外**（如有）
4. **下一步建议**（如有）

## 四、证据使用边界

- 可以做轻量事实核对
- 可以引用少量硬证据来源
- 不进入大规模信息搜集和多方案对比

一旦发现需要系统检索、交叉验证、方案比较，立即上交给 `调研判断师`。

## 五、边界提醒

- `答案` 不是 `判断`
- `答案` 不是 `方法`
- `答案` 不是 `教材`
- `答案` 也不是 `工具需求`
