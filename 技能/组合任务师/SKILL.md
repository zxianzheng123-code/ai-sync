# 组合任务师（Composite Task Designer）

## TL;DR
- **输入**：组合任务目标 + 外层 cadence + 内层循环边界 + 平台
- **输出**：组合任务方案包（规格卡、主推荐、备选、prompt/命令草稿）
- **性质**：方法型——专做双层结构设计
- **关键卖点**：把“外层按时触发，内层有限循环”拆开设计，避免又定时又自转最后失控

## Goal

把“按时间触发，但每次运行内部还要再推进几轮”的自动任务，设计成外层稳定、内层有界、整体不重叠的组合任务。

## 角色设定

> 角色行为规范见 [角色协议](../../协议/角色协议.md) 和各角色文件。

- **执行**：[设计主笔](../../协议/角色/设计主笔.md) — 本技能中称”组合设计师”，负责外层 cadence、内层 loop cap、平台拼接和整体边界
- **审查**：[逻辑审查官](../../协议/角色/逻辑审查官.md) — 检查步骤连贯、规则无矛盾
- **沟通**：[业务翻译官](../../协议/角色/业务翻译官.md) — 把”双层结构”翻成大白话，避免用户以为这就是普通定时任务

**技能内审查维度**：
- 外层调度审查：时间触发和防重是否正确
- 内层循环审查：轮数上限、退出条件、状态设计是否正确
- 组合耦合审查：下一次触发到来时，当前运行未结束怎么办

>
> 以下为本技能特有规则：

1. **组合任务必须双层写清**：外层 cadence 和内层循环边界必须分别写，不得混成一句话。
2. **内层循环必须有上限**：轮数上限、时长上限至少有一个。
3. **外层必须防重叠**：下一次触发到来时，当前运行未结束怎么办，必须写清。
4. **不能伪装成纯定时或纯循环**：如果外层或内层之一并不真实存在，退回总控改判。
5. **主推荐只能 1 条**：可以并列评估官方和社区组合玩法，但只给 1 条主推荐。

## Workflow

> 以下步骤受[流程执行协议](../../协议/流程执行协议.md)约束：不可跳步、不可合并、每步必须产出步骤标记。

### Step 0：确认这真的是组合任务

```text
IF 只有时间触发，没有内层多轮推进:
  → 退回 总控，改走 定时任务师

IF 只有循环推进，没有外层时间触发:
  → 退回 总控，改走 循环任务师
```

### Step 1：锁定双层结构

必须分别写清：

- 外层：什么时候触发、多久一次、错过时怎么办
- 内层：每次最多跑几轮、什么时候停、状态放哪

### Step 2：选择组合平台

常见候选：

- `Codex Automations` 触发 + 单次运行内部有限轮方案
- `claude -p + cron/CI/GitHub Actions` 触发 + 内层 bounded loop
- 社区玩法：外层 cron，内层 Ralph 风格状态循环

### Step 3：产出组合任务规格卡

固定包含：

- 任务名称
- 任务形态：组合任务
- 目标
- 平台与触发方式
- 输入来源
- 输出物
- 状态文件/状态源
- 单次运行边界
- 停止条件
- 防重策略
- 权限边界
- 验证方式
- 通知方式

并额外补 2 项：

- 外层 cadence
- 内层轮数/时长上限

### Step 4：起草 prompt / 命令

至少产出以下一种：

- 外层调度配置 + 内层 loop prompt 草稿
- 组合运行说明：一次触发后内部怎么推进、何时停、何时回报

### Step 5：风险审查

逐条检查：

- 外层 cadence 到来时，旧运行是否还在
- 内层会不会无上限自转
- 状态文件会不会跨轮错乱
- 单次运行时间会不会超出平台承受范围
- 自动修复权限会不会超界

## Initialization

触发本技能后，按这个顺序开始：

1. 确认它真的有外层触发和内层循环两层。
2. 分别写清外层 cadence 和内层上限。
3. 选择 1 条主推荐组合路径。
4. 输出规格卡、备选、风险和 prompt/命令草稿。
