# 执行闭环协议

> 定义所有技能在执行、推进、交付、回看时必须遵守的规则。本协议只管"怎么推进、怎么汇报、怎么验收、怎么复盘"；不再负责定义题目，也不再负责检索或判断。

## 一、启动条件

执行前必须满足：

1. 当前题目已说清
2. 当前要拿到的东西已锁住
3. 成功标志已明确

不满足时，不开工。

## 二、单步推进

执行节奏固定为：

1. 一次只推进 1 个步骤
2. 当前步骤未完成，不开启下一步
3. 每步做完后，先确认结果，再继续

## 三、里程碑汇报

以下时机必须汇报：

- 阶段开始
- 里程碑完成
- 路线需要调整
- 你主动询问

汇报内容固定 3 件事：

1. 现在在做什么
2. 已经做到哪
3. 下一步是什么

## 四、跑偏检测

每完成一个子步骤后自检：

1. 还在服务原目标吗？
2. 有没有不知不觉扩范围？
3. 有没有漏掉原本必须做的事？

发现跑偏了：

> 老板，原目标是 `[X]`，但我发现现在走到了 `[Y]`。我建议改成 `[Z]`，行吗？

## 五、最小改动原则

修改已有内容时：

- 只改必须改的
- 不顺手加功能
- 不顺手重构
- 改完要检查周边没被带坏

## 六、交付与验收

每次交付都必须回答：

1. 交付了什么
2. 证据是什么
3. 是否达到成功标志

标准格式：

```
检查项：...
结果：✅ / ❌
证据：...
```

## 七、遗留问题

执行过程中发现但当前不处理的问题，不能丢：

- 记录问题描述
- 记录发现时机
- 记录建议处理时机
- 交付时主动向你汇报

## 八、复盘触发

以下任一触发复盘：

- 路线明显跑偏了
- 验收不通过
- 你对结果不满意

复盘只回答 4 件事：

1. 哪里出了问题
2. 根因是什么
3. 现在怎么补
4. 下次怎么避免

## 九、边界提醒

- 本协议不定义题目
- 本协议不定义证据质量
- 本协议不定义推荐逻辑
- 本协议只保证"从开始到交付"不走样
