## 执行模式
Master Agent 负责调度和验收，不亲自干活。每个步骤启动 SubAgent 执行。

## 工作人员
- 全栈开发（→ 技能/人设库/全栈开发/人设.md）
- 测试工程师（→ 技能/人设库/测试工程师/人设.md）

## Workflow

### Step 0：加载上下文（Master Agent）
执行 脚本/加载上下文.sh

### Step 1：确认双层结构
- 执行者：全栈开发（SubAgent）
- 目标：确认需求确实有外层时间触发和内层多轮推进两层；如只有一层则退回循环定时任务总控改判
- 成功标准：外层和内层各自独立存在；已分别写清外层（触发时间、频率、错过处理）和内层（最大轮数、停止条件、状态存放）
- 输入：上游（循环定时任务总控）交接信息或用户直接需求
- 输出：双层结构定义（外层cadence + 内层循环边界）

### Step 2：选择组合平台
- 执行者：全栈开发（SubAgent）
- 目标：在 Codex Automations+内部有限轮、claude -p+cron/CI+内层bounded loop、外层cron+内层Ralph风格状态循环中评估并选择1条主推荐
- 成功标准：主推荐只有1条；选择理由明确；外层防重叠方案已设计
- 输入：Step 1 的双层结构定义
- 输出：组合平台主推荐 + 备选方案

### Step 3：产出规格卡和草稿
- 执行者：全栈开发（SubAgent）
- 目标：产出完整的组合任务规格卡 + 外层调度配置 + 内层loop prompt草稿
- 成功标准：规格卡包含全部字段（任务名称、形态、目标、平台与触发方式、输入来源、输出物、状态文件/状态源、单次运行边界、停止条件、防重策略、权限边界、验证方式、通知方式）+ 额外的外层cadence和内层轮数/时长上限
- 输入：Step 1 + Step 2 的结果
- 输出：规格卡 + 外层调度配置 + 内层prompt草稿

### Step 4：风险审查
- 执行者：测试工程师（SubAgent）
- 目标：逐条检查风险——外层cadence到来时旧运行是否还在、内层无上限自转、状态文件跨轮错乱、单次运行超出平台承受范围、自动修复权限超界
- 成功标准：所有风险项已检查；发现的风险已给出缓解方案
- 输入：Step 3 的规格卡和草稿
- 输出：风险清单 + 缓解方案

### Step 5：验证
- 如有 脚本/验证/ 下的脚本 → 自动执行
- 如有 作业指引/交付自查清单.md → 逐项检查
- 额外检查项：
  - [ ] 外层cadence和内层循环边界分别写清
  - [ ] 内层循环有上限（轮数或时长）
  - [ ] 外层防重叠方案已设计
  - [ ] 双层结构不是伪装的纯定时或纯循环
  - [ ] 风险清单完整
- 全部通过才进入交付

### Step 6：交付审核（Master Agent）
输出组合任务方案包（规格卡+主推荐+备选+风险+外层调度配置+内层prompt草稿），等用户确认。如由总控调度则回传给总控。
