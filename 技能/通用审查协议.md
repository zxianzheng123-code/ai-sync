# 通用审查协议

> 本文件定义多个技能共用的审查规则与互撕循环模板。各 SKILL.md 引用本协议，不重复写入。

## 通用审查规则

1. **互撕硬门禁**：核心互撕至少 5 回合，其余互撕至少 3 回合；未到回合数不得通过。
2. **三签门禁**：互撕阶段每轮都必须全部审查专家明确写出"通过"，否则不得进入下一步。
3. **Lead 单次采纳**：Lead 每轮只采纳 1 条审查建议进行修订。
4. **指名道姓**：审查意见必须指出具体页面/组件/元素/文件名称，禁止笼统评价。
5. **必须找出问题**：每回合每位专家至少指出 1 个具体问题。若确实无问题，必须列出检查项证明。
6. **引用证据**：通过/不通过必须附带可验证证据。
7. **大白话沟通**：面向客户的沟通禁止技术黑话。客户经理负责翻译。

## 互撕循环模板

所有互撕阶段统一遵循以下流程，各技能指定回合数和签字角色即可：

```
回合 1（必做）：Lead 提交初稿/草案 → 各专家指出 ≥1 个具体问题（指名道姓 + 引用证据）
回合 2..N-1（必做）：Lead 采纳 1 条修订 → 各专家指出 ≥1 个具体问题
回合 N（必做）：Lead 采纳 1 条修订
终审回合：各专家终审（指名道姓 + 引用证据）：
  - 仍有问题 → 回到回合 N
  - 全签通过（附检查清单），结束循环

每回合输出签字格式：
  [专家角色名 签字] 通过/不通过：<引用具体位置 + 理由>
```
