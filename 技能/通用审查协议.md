# 通用审查协议

> 本文件定义多个技能共用的审查规则与交叉检查循环模板。各 SKILL.md 引用本协议，不重复写入。

## 通用审查规则

1. **最低检查轮数**：重要内容至少检查 5 轮，其余至少 3 轮；未到轮数不得通过。为什么要这么多轮？因为检查越仔细，交到你手上的东西越靠谱。
2. **全员确认**：检查阶段每轮都必须全部审查专家明确写出"通过"，否则不得进入下一步。
3. **主笔单次采纳**：主笔每轮只采纳 1 条审查建议进行修订。这样改一条、查一条，不容易改出新问题。
4. **指名道姓**：审查意见必须指出具体页面/组件/元素/文件名称，禁止笼统评价。
5. **必须找出问题**：每回合每位专家至少指出 1 个具体问题。若确实无问题，必须列出完整检查项清单（逐条说明检查了什么、结论是什么），证明不是没查而是真没问题。
6. **引用证据**：通过/不通过必须附带可验证证据。
7. **大白话沟通**：面向你的一切沟通禁止技术黑话。客户经理负责翻译。

## 交叉检查循环模板

所有检查阶段统一遵循以下流程，各技能指定轮数和确认角色即可：

```
第 1 轮（必做）：主笔提交初稿/草案 → 各专家指出 ≥1 个具体问题（指名道姓 + 引用证据）
第 2..N-1 轮（必做）：主笔采纳 1 条修订 → 各专家指出 ≥1 个具体问题
第 N 轮（必做）：主笔采纳 1 条修订
终审轮：各专家终审（指名道姓 + 引用证据）：
  - 仍有问题 → 回到第 N 轮
  - 全员确认通过（附检查清单），结束循环

每轮输出确认格式：
  [专家角色名 确认] 通过/不通过：<引用具体位置 + 理由>
```
