# 技能依赖图

> 本文件定义工作流程间的数据流向与协作关系。修改任何工作流的输入/输出时，须同步更新本图。

## 技能分组总览

```
┌─────────────────────────────────────────────────┐
│  用户入口                                        │
│  客服工作（唯一入口）                              │
├─────────────────────────────────────────────────┤
│  转化能力                                        │
│  问题拆解（前处理）                               │
│  答案澄清（答案终点）                             │
│  调研判断（判断终点）                             │
│  方法提炼（方法终点）                             │
│  教材编写（教材终点）                             │
├─────────────────────────────────────────────────┤
│  造工具                                          │
│  造工具（总调度）                                 │
│  → 需求规划 → 产品逻辑 → 产品设计 → 全栈编码     │
│  → 部署上线 ← 代码维护                           │
├─────────────────────────────────────────────────┤
│  自动化                                          │
│  循环定时任务（总调度）                            │
│  → 循环任务 / 定时任务 / 组合任务                  │
├─────────────────────────────────────────────────┤
│  造技能                                          │
│  Skill编译 / 全景图生成                           │
├─────────────────────────────────────────────────┤
│  质量保障                                        │
│  测试循环 / Codex对等审查                         │
├─────────────────────────────────────────────────┤
│  万能兜底                                        │
│  通用任务                                        │
└─────────────────────────────────────────────────┘
```

## 数据流拓扑

```
用户
 │
 ▼
客服工作（接待 → 成题 → 路由）
 │
 ├─ 题目还乱 ──→ 问题拆解 ──→ 拆解清单 + 推荐终点
 │                   │
 │                   ├──→ 答案澄清
 │                   ├──→ 调研判断
 │                   ├──→ 方法提炼
 │                   ├──→ 教材编写
 │                   └──→ 造工具
 │
 ├─ 要解释/澄清 ──→ 答案澄清 ──→ 答案卡
 │
 ├─ 要对比/诊断/选型 ──→ 调研判断 ──→ 判断报告
 │
 ├─ 要固化 SOP/规则 ──→ 方法提炼 ──→ 方法文档
 │
 ├─ 要沉淀课本/教材 ──→ 教材编写 ──→ 教材文档
 │
 ├─ 要做成工具/系统 ──→ 造工具（总调度 → progress.md）
 │      │
 │      ▼
 │   需求规划 ──→ project_context.md
 │      │
 │      ▼
 │   产品逻辑 ──→ product_logic.md
 │      │
 │      ├────────────────────┐
 │      ▼                    ▼
 │   产品设计            全栈编码 Phase A-B
 │   → design_spec.md      → infra_spec.md
 │   → interaction_map.md   → code_blueprint.md
 │   → 可点击原型          → 干净项目骨架
 │      │                    │
 │      └────────┬───────────┘
 │               ▼
 │      全栈编码 Phase C-E ──→ 生产级前端代码 + 生产级后端代码
 │               │
 │               ▼
 │         部署上线 ──→ 线上可访问的产品 + deploy_config.md + 验收签收记录
 │               │
 │               └─→ 后续维护 ──→ 代码维护
 │
 ├─ 要自动化任务 ──→ 循环定时任务 ──→ 循环/定时/组合任务
 │
 ├─ 要造新技能 ──→ Skill编译
 │
 ├─ 要全面体检 ──→ 测试循环
 │
 ├─ 要跨模型审查 ──→ Codex对等审查 ──→ 审查报告
 │
 └─ 其他任务 ──→ 通用任务（兜底）/ 计划模式（临时编排）
```

## 人设引用关系

| 人设 | 模型 | 被引用的工作流 |
|---|---|---|
| 路由调度员 | Opus 4.6 · max | 客服工作, 造工具, 循环定时任务 |
| 产品 | Opus 4.6 · max | 问题拆解, 需求规划, 产品逻辑, Skill编译, 全景图生成 |
| UI设计 | Opus 4.6 · max | 产品设计 |
| 全栈开发 | Opus 4.6 · max | 全栈编码, 部署上线, 代码维护, 循环任务, 定时任务, 组合任务, 测试循环 |
| 测试工程师 | Opus 4.6 · max | 全栈编码, 代码维护, 循环任务, 定时任务, 组合任务, 测试循环 |
| 写作主笔 | Opus 4.6 · max | 答案澄清, 调研判断, 方法提炼, 教材编写, 产品设计, Skill编译 |
| 审查官 | GPT-5.4 · xhigh | 答案澄清, 调研判断, 方法提炼, 教材编写, 问题拆解, 全栈编码, 代码维护, Codex对等审查 |
