# 技能依赖图

> 本文件定义技能间的数据流向与协作关系。修改任何技能的输入/输出时，须同步更新本图。

## 技能分组总览

```
┌─────────────────────────────────────────────────┐
│  转化能力                                        │
│  总转化器（默认入口）                              │
│  问题拆解师（前处理）                              │
│  调研判断师（判断终点）                            │
│  方法提炼师（方法终点）                            │
│  教材编写师（教材终点）                            │
├─────────────────────────────────────────────────┤
│  造工具                                            │
│  造工具总控（唯一入口）                              │
│  → 需求规划师 → 产品设计师 → 全栈编码师             │
│  → 部署上线师 ← 代码维护师                         │
├─────────────────────────────────────────────────┤
│  造技能                                          │
│  Skill编译器 │ 通用审查协议                        │
├─────────────────────────────────────────────────┤
│  万能兜底                                        │
│  通用任务引擎                                     │
└─────────────────────────────────────────────────┘
```

## 转化工作流（数据流拓扑）

```
老板原始输入
     │
     ├─ 题目还乱 / 目标混杂 ──→ 总转化器 ──→ 问题拆解师 ──→ 拆解清单 + 推荐终点
     │                                          │
     │                                          ├──→ 调研判断师
     │                                          ├──→ 方法提炼师
     │                                          ├──→ 教材编写师
     │                                          └──→ 造工具总控
     │
     ├─ 要解释/澄清 ──→ 总转化器 ──→ 答案
     │
     ├─ 要对比/诊断/选型 ──→ 调研判断师 ──→ 判断报告
     │
     ├─ 要固化 SOP/规则/流程 ──→ 方法提炼师 ──→ 方法文档
     │
     ├─ 要沉淀课本/训练资料 ──→ 教材编写师 ──→ 教材文档
     │
     ├─ 要做成工具/系统 ──→ 造工具总控：
     │      │
     │      ▼
     │   需求规划师 ──→ project_context.md
     │      │
     │      ├──────────────┐
     │      ▼              ▼
     │   产品设计师     全栈编码师 Phase A
     │      │              │
     │      │ 设计产物      │ infra_spec.md + 项目骨架
     │      │              │
     │      └──────┬───────┘
     │             ▼
     │      全栈编码师 Phase B-E ──→ 生产代码
     │             │
     │             ▼
     │         部署上线师 ──→ 线上可访问的工具
     │             │
     │             └─→ 后续维护 ──→ 代码维护师
     │
     └─ 其他任务 ──→ 通用任务引擎
```

## 产出物注册表

| 产出物 | 生产技能 | 消费技能 | 路径 |
|--------|---------|---------|------|
| `拆解清单` | 问题拆解师 | 调研判断师、方法提炼师、教材编写师、造工具总控 | 会话内上下文 |
| `判断报告` | 调研判断师 | 方法提炼师、教材编写师 | 会话内上下文 / 文档 |
| `[主题]_方法.md` | 方法提炼师 | 人、AI、造工具总控 | 业务文档目录 |
| `[主题]_教材.md` | 教材编写师 | 人、造工具总控 | 业务文档目录 |
| `project_context.md` | 需求规划师 | 产品设计师、全栈编码师 | `design/` |
| `layout_blueprint.md` | 产品设计师 | 全栈编码师 | `design/` |
| `style_guidebook.pen` | 产品设计师 | 全栈编码师 | `design/` |
| `static_ui_mock.pen` | 产品设计师 | 全栈编码师 | `design/` |
| `interaction_map.md` | 产品设计师 | 全栈编码师 | `design/` |
| `infra_spec.md` | 全栈编码师 Phase A | 全栈编码师 Phase B-E、部署上线师、代码维护师 | 项目根目录 |
| `code_blueprint.md` | 全栈编码师 Phase B | 全栈编码师 Phase C-E、部署上线师 | `design/` |
| `deploy_config.md` | 部署上线师 | （终端产物） | 项目根目录 |

## 独立技能（无造工具链数据依赖）

| 技能 | 层级 | 输入 | 输出 |
|------|------|------|------|
| 总转化器 | 转化入口 | 任意原始输入 | `答案` / 下游路由 |
| 问题拆解师 | 前处理 | 任何复杂想法/问题/需求 | 拆解清单 + 推荐终点 |
| 调研判断师 | 判断终点 | 已成形的判断题 + 可选拆解清单 | 判断报告 |
| 方法提炼师 | 方法终点 | 已成形的方法目标 + 可选判断结论 | 方法文档 |
| 教材编写师 | 教材终点 | 已成形的教材目标 + 可选判断结论 | 教材文档 |
| 代码维护师 | 造工具链 | 任意代码 + 可选 infra_spec | 修复/重构/优化/测试后的代码 |
| Skill编译器 | 元技能 | 教材/方法文档 + 技能需求 | SKILL.md |
| 通用任务引擎 | 万能入口 | 任意任务描述 | 动态交付物 |
