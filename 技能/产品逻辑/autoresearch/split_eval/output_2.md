# Product Logic

- 页面设计必须覆盖本文档中的所有场景和联动点。

- 团购必须在截止时间前达到最低成团人数，否则自动取消并退款

## Business Scenarios

### Scenario: 消费者注册与团长认证
- Trigger: 消费者进入用户管理模块，账号状态为"未注册"时触发
- Flow:
  1. 消费者 在 用户管理 填写注册信息并提交 → 系统将账号状态设为"已注册"，写入注册时间
  2. 消费者 在 用户管理 提交团长认证申请 → 系统将认证申请状态设为"审核中"，写入申请时间
  3. 系统 在 用户管理 审核通过认证申请 → 系统将认证申请状态设为"已认证"，账号角色更新为"团长"
- Success End: 账号角色为"团长"，认证申请状态为"已认证"
- Exception Paths:
  - IF 注册信息填写不完整（必填项为空）: 系统将注册操作状态设为"提交失败"，向消费者展示"请补充必填信息后重新提交"，消费者可补填后重新提交。
  - IF 团长认证申请审核不通过: 系统将认证申请状态设为"审核未通过"，向消费者展示"团长认证未通过，请检查材料后重新申请"，消费者可修改材料后重新提交认证申请。

### Scenario: 团长发起团购
- Trigger: 团长认证状态为"已认证"，团长进入团购发起模块选品时触发
- Flow:
  1. 团长 在 团购发起 选择商品并填写团购价 → 系统将选中商品编号和团购价写入团购草稿
  2. 团长 在 团购发起 填写最低成团人数和截止时间 → 系统将最低成团人数和截止时间写入团购草稿
  3. 团长 在 团购发起 提交发布 → 系统将团购状态设为"拼单中"，写入发布时间
- Success End: 团购状态为"拼单中"
- Exception Paths:
  - IF 截止时间早于当前时间: 系统将截止时间字段标记为"无效"，向团长展示"截止时间须晚于当前时间，请重新设置"，团长可修改截止时间后重新提交。
  - IF 最低成团人数未填写: 系统将发布操作状态设为"信息不完整"，向团长展示"请填写最低成团人数后再发布"，团长可补填后重新提交。

### Scenario: 消费者加入拼单
- Trigger: 消费者进入拼单管理模块，团购状态为"拼单中"时触发
- Flow:
  1. 消费者 在 拼单管理 选择目标团购并提交加入 → 系统将该消费者拼单记录状态设为"已支付"，已参团人数加 1，写入参团时间
  2. 系统 在 拼单管理 检测已参团人数是否达到最低成团人数 → 若已达到，将团购状态设为"已成团"，写入成团时间
- Success End: 团购状态为"已成团"
- Exception Paths:
  - IF 截止时间到达时已参团人数未达到最低成团人数（团购必须在截止时间前达到最低成团人数，否则自动取消并退款）: 系统将团购状态设为"已取消"，对所有已支付消费者发起退款，将各拼单记录退款状态设为"已退款"，向消费者展示"本次团购未成团，款项将原路退回"，消费者可等待退款到账。
  - IF 消费者提交加入时团购截止时间已过: 系统将加入操作状态设为"不可用"，向消费者展示"该团购已截止，无法加入"，消费者可返回拼单管理列表选择其他团购。

### Scenario: 供应商发货与团长确认收货
- Trigger: 供应商进入配送管理模块，团购状态为"已成团"时触发
- Flow:
  1. 供应商 在 配送管理 查看待发货列表 → 系统展示所有状态为"已成团"的团购及收货地址
  2. 供应商 在 配送管理 提交发货并填写物流信息 → 系统将配送状态设为"已发货"，写入发货时间和物流信息
  3. 团长 在 配送管理 确认收货 → 系统将配送状态设为"团长已收货"，写入收货时间
  4. 团长 在 配送管理 通知消费者自提 → 系统将各消费者拼单自提状态设为"待自提"，向消费者展示"货物已到，请在 3 天内自提"
- Success End: 配送状态为"团长已收货"，消费者拼单自提状态为"待自提"
- Exception Paths:
  - IF 供应商填写物流信息时运单号为空: 系统将发货操作状态设为"提交失败"，向供应商展示"请填写物流运单号后再提交"，供应商可补填运单号后重新提交。
  - IF 团长确认收货时发现货损（商品损坏或数量不符）: 系统将配送状态设为"货损待处理"，向供应商展示"团长反馈货损，请联系处理"，团长可在配送管理提交货损记录等待供应商响应。

### Scenario: 消费者自提
- Trigger: 消费者进入配送管理模块，消费者拼单自提状态为"待自提"时触发
- Flow:
  1. 消费者 在 配送管理 查看自提信息并前往自提点取货 → 系统展示自提地址和取货码
  2. 消费者 在 配送管理 出示取货码完成自提 → 系统将该消费者拼单自提状态设为"已自提"，写入自提时间
- Success End: 消费者拼单自提状态为"已自提"
- Exception Paths:
  - IF 消费者提供的取货码与系统记录不匹配: 系统将本次核验状态设为"验证失败"，向消费者展示"取货码不正确，请核实后重试"，消费者可重新查看配送管理中的取货码后再次出示。
  - IF 消费者取货时发现货物缺损: 系统将该消费者拼单自提状态设为"异常待处理"，向消费者展示"请联系团长处理缺损问题"，消费者可在配送管理提交缺损记录等待团长响应。

### Scenario: 平台结算团长佣金与供应商货款
- Trigger: 该团购所有消费者拼单自提状态均为"已自提"或"自动确认"，系统进入结算管理触发结算时触发
- Flow:
  1. 系统 在 结算管理 汇总该团购实际成交金额 → 系统写入实际成交金额，将结算状态设为"计算中"
  2. 系统 在 结算管理 按实际成交金额的 10% 计算团长佣金 → 系统写入团长佣金金额，将团长佣金结算状态设为"待确认"
  3. 系统 在 结算管理 计算供应商货款（实际成交金额扣除平台抽成和团长佣金） → 系统写入供应商货款金额，将供应商货款结算状态设为"待确认"
  4. 团长 在 结算管理 确认佣金金额 → 系统将团长佣金结算状态设为"已确认"
  5. 供应商 在 结算管理 确认货款金额 → 系统将供应商货款结算状态设为"已确认"，整体结算状态设为"已完成"，写入结算完成时间
- Success End: 团长佣金结算状态为"已确认"，供应商货款结算状态为"已确认"，整体结算状态为"已完成"
- Exception Paths:
  - IF 团长收款账户信息未绑定: 系统将团长佣金结算状态设为"待绑定账户"，向团长展示"请先在用户管理中绑定收款账户"，团长可进入用户管理补充收款信息后返回确认。
  - IF 供应商货款结算金额与供应商预期不符: 系统将供应商货款结算状态设为"争议中"，向供应商展示"结算金额存在差异，已提交复核"，供应商可等待平台人工复核结果。

## Module Connections

| 上游模块 | 触发操作 | 下游模块 | 自动反应 |
|---------|---------|---------|---------|
| 用户管理 | 账号角色更新为"团长" | 团购发起 | 系统将该账号团购发布权限状态设为"已开放"，团购发起模块展示发布入口 |
| 团购发起 | 团购状态设为"拼单中" | 拼单管理 | 系统将该团购信息写入拼单管理列表，消费者可见并可加入 |
| 拼单管理 | 已参团人数达到最低成团人数，团购状态设为"已成团" | 配送管理 | 系统将该团购写入供应商待发货列表，配送状态设为"待发货" |
| 拼单管理 | 截止时间到达且已参团人数未达最低成团人数，团购状态设为"已取消" | 结算管理 | 系统将该团购各拼单退款记录写入结算管理，退款状态设为"待退款" |
| 配送管理 | 全部消费者拼单自提状态均为"已自提"或"自动确认" | 结算管理 | 系统将该团购结算解锁，写入可结算金额，将结算状态设为"计算中" |
| 结算管理 | 整体结算状态设为"已完成" | 用户管理 | 系统将团长和供应商的历史结算记录写入各自账号档案 |

## Trigger Rules

| 编号 | 触发条件 | 触发动作 | 通知对象 |
|------|---------|---------|---------|
| T1 | 系统定时检测：团购截止时间到达且已参团人数未达最低成团人数 | 系统将团购状态设为"已取消"，对所有已支付消费者发起退款，将各拼单退款状态设为"已退款" | 消费者 |
| T2 | 系统定时检测：配送状态为"已发货"且自发货时间起超过 48 小时，团长未确认收货 | 系统将收货催促提醒状态设为"已发送"，向团长推送"供应商发货已超 48 小时，请尽快确认收货" | 团长 |
| T3 | 系统定时检测：消费者拼单自提状态为"待自提"且自团长通知时间起超过 3 天 | 系统将该消费者拼单自提状态设为"自动确认"，写入自动确认时间 | 无 |
