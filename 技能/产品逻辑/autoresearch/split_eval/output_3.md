# Product Logic

## Instructions For Design & Coding AI

- 页面设计必须覆盖本文档中的所有场景和联动点。

- 发现逻辑矛盾或缺口时，以本文档为准，向上游反馈。
- 预约必须提前至少 24 小时，节假日提前 72 小时
- 宠物必须有 30 天内的疫苗接种记录才能入住
- 取消预约提前 48 小时全额退定金，否则扣 50%
- 寄养期间每天至少上传 1 张宠物照片给主人

## Business Scenarios

### Scenario: 宠物主人提交预约并支付定金
- Trigger: 宠物主人进入预约管理模块，选择寄养日期和房间类型后提交预约时触发
- Flow:
  1. 宠物主人 在 宠物档案 确认宠物疫苗接种记录 → 系统返回该宠物最近一次疫苗接种日期及距今天数
  2. 宠物主人 在 预约管理 选择寄养日期、房间类型并提交预约申请 → 系统校验所选日期距当前时间是否满足提前时限，预约状态设为"待支付"，写入提交时间
  3. 宠物主人 在 支付管理 完成定金支付 → 系统将定金支付状态设为"已付定金"，写入定金支付时间，预约状态设为"已预约"
  4. 系统 在 通知管理 推送预约确认通知 → 向宠物主人发送"预约已提交，等待店长确认"通知，通知状态设为"已推送"
- Success End: 预约状态为"已预约"，定金支付状态为"已付定金"
- Exception Paths:
  - IF 所选寄养日期为普通日且距当前时间不足 24 小时（预约必须提前至少 24 小时）: 系统将预约草稿状态设为"时间不足"，向宠物主人展示"预约须至少提前 24 小时，请重新选择日期"，宠物主人可修改日期后重新提交。
  - IF 所选寄养日期为节假日且距当前时间不足 72 小时（节假日提前 72 小时）: 系统将预约草稿状态设为"时间不足"，向宠物主人展示"节假日预约须至少提前 72 小时，请重新选择日期"，宠物主人可修改日期后重新提交。
  - IF 宠物疫苗接种记录距当前时间超过 30 天（宠物必须有 30 天内的疫苗接种记录才能入住）: 系统将预约草稿状态设为"疫苗不符"，向宠物主人展示"宠物须有 30 天内的疫苗接种记录才能入住，请先更新疫苗记录"，宠物主人可前往宠物档案更新疫苗记录后重新提交。

### Scenario: 店长确认宠物入住
- Trigger: 预约状态为"已预约"且到达入住日期，店长进入预约管理模块执行入住确认操作时触发
- Flow:
  1. 店长 在 预约管理 查看当日待入住预约列表 → 系统展示所有状态为"已预约"且入住日期为当日的预约记录
  2. 店长 在 宠物档案 核查目标宠物疫苗接种记录 → 系统展示该宠物最近一次疫苗接种日期及距今天数
  3. 店长 在 预约管理 确认宠物入住并分配房间 → 系统将预约状态设为"已入住"，写入实际入住时间和分配房间编号
- Success End: 预约状态为"已入住"
- Exception Paths:
  - IF 宠物疫苗接种记录距入住日期超过 30 天（宠物必须有 30 天内的疫苗接种记录才能入住）: 系统将入住确认操作状态设为"已拦截"，向店长展示"该宠物疫苗记录不符合 30 天要求，无法办理入住"，店长可通知宠物主人补充更新疫苗记录后重新操作。
  - IF 所选房间类型当日无空余: 系统将入住确认操作状态设为"房间不足"，向店长展示"所选房间类型当日已满，请调整房间安排"，店长可为宠物主人调整为其他可用房间类型后重新确认。

### Scenario: 寄养员记录每日寄养日志并上传照片
- Trigger: 预约状态为"已入住"，寄养员进入寄养日志模块为当日寄养宠物填写日志时触发
- Flow:
  1. 寄养员 在 寄养日志 选择目标宠物的当日日志条目 → 系统展示该宠物当日日志填写表单（喂食记录、活动记录、健康观察、照片上传）
  2. 寄养员 在 寄养日志 填写喂食记录、活动记录和健康观察 → 系统将填写内容写入当日日志草稿，日志状态设为"草稿"
  3. 寄养员 在 寄养日志 上传当日宠物照片（至少 1 张）→ 系统将照片文件写入当日日志，当日照片上传数量加 1
  4. 寄养员 在 寄养日志 提交当日日志 → 系统将日志状态设为"已提交"，写入提交时间
  5. 系统 在 通知管理 将当日日志（含照片）打包为日报并推送 → 向宠物主人发送每日寄养报告，推送状态设为"已推送"
- Success End: 日志状态为"已提交"，每日寄养报告推送状态为"已推送"
- Exception Paths:
  - IF 提交时当日照片上传数量为 0（寄养期间每天至少上传 1 张宠物照片给主人）: 系统将日志提交操作状态设为"照片缺失"，向寄养员展示"每天须至少上传 1 张宠物照片，请先上传照片后再提交"，寄养员可上传照片后重新提交。
  - IF 上传文件格式不符合图片格式要求: 系统将该文件上传状态设为"格式不支持"，向寄养员展示"文件格式不支持，请上传图片格式文件"，寄养员可重新选择正确格式的文件上传。

### Scenario: 宠物主人取消预约并申请退定金
- Trigger: 预约状态为"已预约"，宠物主人进入预约管理模块选择取消预约时触发
- Flow:
  1. 宠物主人 在 预约管理 选中目标预约并提交取消申请 → 系统记录取消申请时间，计算距入住日期的剩余时间，预约状态设为"取消处理中"
  2. 系统 在 支付管理 根据距入住日期剩余时间计算退款金额 → 距入住时间超过 48 小时则退款金额设为定金全额；距入住时间不超过 48 小时则退款金额设为定金的 50%，退款单状态设为"待退款"
  3. 系统 在 支付管理 执行退款 → 退款单状态设为"已退款"，写入退款完成时间
  4. 系统 在 预约管理 更新预约状态 → 预约状态设为"已取消"
  5. 系统 在 通知管理 推送取消确认通知 → 向宠物主人发送"预约已取消，退款 [金额] 已处理"通知，通知状态设为"已推送"
- Success End: 预约状态为"已取消"，退款单状态为"已退款"
- Exception Paths:
  - IF 取消时距入住日期不超过 48 小时（取消预约提前 48 小时全额退定金，否则扣 50%）: 系统将退款金额设为定金的 50%，向宠物主人展示"距入住不足 48 小时，定金将扣除 50%，请确认是否继续取消"，宠物主人可确认继续或放弃取消。
  - IF 退款执行失败: 系统将退款单状态设为"退款失败"，向宠物主人展示"退款处理失败，请联系店长处理"，宠物主人可联系店长重新发起退款。

### Scenario: 宠物主人支付尾款并完成接宠
- Trigger: 预约状态为"已入住"且到达寄养结束日期，宠物主人进入支付管理模块完成尾款支付时触发
- Flow:
  1. 系统 在 支付管理 生成尾款账单 → 系统根据实际寄养天数计算尾款金额，将账单状态设为"待支付"
  2. 系统 在 通知管理 推送接宠提醒 → 向宠物主人发送"寄养结束，请支付尾款并安排接宠"通知，通知状态设为"已推送"
  3. 宠物主人 在 支付管理 完成尾款支付 → 系统将尾款支付状态设为"已付尾款"，写入尾款支付时间
  4. 店长 在 预约管理 确认宠物已接回 → 系统将预约状态设为"已完成"，写入接宠确认时间
- Success End: 预约状态为"已完成"，尾款支付状态为"已付尾款"
- Exception Paths:
  - IF 尾款支付失败（如余额不足）: 系统将尾款支付状态设为"支付失败"，向宠物主人展示"支付失败，请检查支付方式后重试"，宠物主人可更换支付方式后重新支付。
  - IF 宠物主人逾期未支付尾款: 系统将账单状态设为"逾期待支付"，向宠物主人展示"尾款已逾期，请尽快完成支付"，宠物主人可进入支付管理完成支付。

## Module Connections

| 上游模块 | 触发操作 | 下游模块 | 自动反应 |
|---------|---------|---------|---------|
| 宠物档案 | 宠物主人确认疫苗接种记录，系统返回接种日期 | 预约管理 | 预约管理读取疫苗接种日期，校验是否在 30 天内，不符合则将预约草稿状态设为"疫苗不符" |
| 预约管理 | 预约状态设为"已预约"（定金支付完成后） | 通知管理 | 通知管理推送预约确认通知给宠物主人，通知状态设为"已推送" |
| 预约管理 | 预约状态设为"取消处理中" | 支付管理 | 支付管理根据距入住日期剩余时间计算退款金额，生成退款单，退款单状态设为"待退款" |
| 支付管理 | 退款单状态设为"已退款" | 预约管理 | 预约管理将预约状态设为"已取消" |
| 预约管理 | 预约状态设为"已取消" | 通知管理 | 通知管理推送取消确认及退款金额通知给宠物主人，通知状态设为"已推送" |
| 寄养日志 | 日志状态设为"已提交" | 通知管理 | 通知管理将当日日志（含照片）打包为日报，向宠物主人推送，推送状态设为"已推送" |
| 预约管理 | 到达寄养结束日期且预约状态为"已入住" | 支付管理 | 支付管理根据实际寄养天数生成尾款账单，账单状态设为"待支付" |
| 支付管理 | 尾款账单状态设为"待支付" | 通知管理 | 通知管理推送接宠提醒通知给宠物主人，通知状态设为"已推送" |
