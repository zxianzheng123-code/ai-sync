# Product Logic

## Instructions For Design & Coding AI
- 本文档定义系统运转逻辑：场景、流程、联动、触发规则。
- 页面设计必须覆盖本文档中的所有场景和联动点。
- 编码时的数据流和事件处理必须与本文档的触发规则对齐。
- 发现逻辑矛盾或缺口时，以本文档为准，向上游反馈。
- 会员等级按年度累计消费划分：银卡（0-2000）、金卡（2000-5000）、钻卡（5000+）
- 积分 1 元 = 1 分，100 分可抵扣 1 元，积分每年 12 月 31 日清零
- 优惠券不可叠加使用，每单最多用 1 张
- 会员连续 6 个月无消费自动降级

## Business Scenarios

### Scenario: 店员为会员办理消费结算
- Trigger: 店员在消费记录模块发起新笔消费，会员已在会员管理中存在有效档案时触发
- Flow:
  1. 店员在会员管理输入会员手机号 → 系统在会员管理中查询并返回该会员的等级、年度累计消费额
  2. 店员在消费记录输入本次消费金额并提交 → 系统在消费记录中将消费明细写入，状态设为"已记录"
  3. 系统在积分管理按消费金额（1元=1分）计算新增积分并累加至该会员积分余额 → 系统将积分余额更新为新值
  4. 系统在会员管理将本次消费金额累加至年度累计消费额 → 系统更新年度累计消费额
  5. 系统在会员管理检测更新后的年度累计消费额是否达到更高等级阈值（2000/5000） → 系统记录检测结果
  6. 系统在会员管理根据检测结果将会员等级更新为对应等级（银卡/金卡/钻卡） → 系统将等级字段写为新值
- Success End: 消费记录状态为"已记录"；积分余额状态为"已更新"；会员等级状态为当前最新等级（银卡/金卡/钻卡）
- Exception Paths:
  - IF 店员输入的手机号在会员管理中不存在：系统将查询状态设为"未找到"，向店员展示"该手机号未注册会员，请核实后重试或引导顾客注册"，店员可重新输入手机号或跳转注册流程。
  - IF 店员输入的消费金额为0或负数：系统将提交状态设为"已拦截"，向店员展示"消费金额须大于0，请重新输入"，店员可重新填写金额后提交。

### Scenario: 店员为新顾客注册会员
- Trigger: 店员在会员管理模块点击"新增会员"时触发
- Flow:
  1. 店员在会员管理填写顾客手机号和姓名 → 系统在会员管理校验该手机号是否已存在，返回校验结果
  2. 店员在会员管理确认提交注册 → 系统在会员管理创建会员档案，将会员状态设为"有效"，初始等级设为"银卡"，年度累计消费额设为0
- Success End: 会员状态为"有效"；会员初始等级为"银卡"
- Exception Paths:
  - IF 店员输入的手机号已被其他会员档案占用：系统将注册状态设为"手机号重复"，向店员展示"该手机号已注册，请核实后重新输入"，店员可修改手机号后重新提交。
  - IF 店员提交时姓名字段为空：系统将提交状态设为"信息不完整"，向店员展示"姓名不得为空，请补充后重新提交"，店员可填写姓名后重新提交。

### Scenario: 会员查询积分并兑换抵扣
- Trigger: 会员在积分管理模块查看积分余额时触发
- Flow:
  1. 会员在积分管理发起查询 → 系统在积分管理中读取该会员当前积分余额并展示，积分查询结果状态设为"已展示"
  2. 会员在积分管理输入本次兑换抵扣的积分数量并提交 → 系统在积分管理校验积分余额是否足够（100分=1元），返回校验结果
  3. 系统在积分管理扣减对应积分 → 系统将积分余额更新为扣减后的新值，兑换记录状态设为"已兑换"
- Success End: 积分查询结果状态为"已展示"；兑换记录状态为"已兑换"
- Exception Paths:
  - IF 积分余额不足100分（不满足最低兑换单位）：系统将兑换请求状态设为"已拦截"，向会员展示"积分不足100分，暂无法兑换"，会员可关闭页面继续消费积累积分。
  - IF 会员输入的兑换积分数量超过当前余额：系统将兑换请求状态设为"余额不足"，向会员展示"当前积分余额不足，请修改兑换数量后重新提交"，会员可调整数量后重新提交。

### Scenario: 会员使用优惠券消费
- Trigger: 会员在优惠券管理中查看可用优惠券列表，且优惠券状态为"已发放"时触发
- Flow:
  1. 会员在优惠券管理查看可用优惠券列表 → 系统在优惠券管理中展示该会员名下状态为"已发放"且未过期的优惠券，查询结果状态设为"已展示"
  2. 会员在优惠券管理选择一张优惠券并确认使用 → 系统在优惠券管理校验所选优惠券的有效期和使用状态，返回校验结果
  3. 系统在优惠券管理将所选优惠券状态设为"使用中" → 系统记录选券时间戳
  4. 店员在消费记录录入本次实付金额（含优惠券抵扣信息）并提交 → 系统在消费记录中写入消费明细（含优惠券抵扣字段），状态设为"已记录"
  5. 系统在优惠券管理将该优惠券状态设为"已使用" → 系统记录核销时间戳
  6. 系统在积分管理按实付金额（扣减优惠券后，1元=1分）累加积分 → 系统将积分余额更新为累加后的新值
  7. 系统在会员管理将实付金额累加至年度累计消费额 → 系统更新年度累计消费额
- Success End: 优惠券状态为"已使用"；消费记录状态为"已记录"；积分余额状态为"已更新"
- Exception Paths:
  - IF 会员在同一笔消费中已选一张优惠券后再次尝试选择第二张：系统将第二次选择状态设为"已拦截"，向会员展示"每单最多使用1张优惠券"，会员可取消当前选择或放弃换券。
  - IF 会员选择的优惠券已过期：系统将该优惠券状态设为"已回收"，向会员展示"该优惠券已过期，已自动回收"，会员可选择其他有效优惠券或不使用优惠券。

### Scenario: 店长配置并发放优惠券
- Trigger: 店长在优惠券管理模块点击"新建优惠券"时触发
- Flow:
  1. 店长在优惠券管理填写优惠券面额、有效期、发放对象等级范围并提交 → 系统在优惠券管理创建优惠券记录，状态设为"待发放"
  2. 店长在优惠券管理点击确认发放 → 系统在会员管理查询符合所选等级范围的会员列表，返回匹配结果
  3. 系统在优惠券管理将优惠券绑定至符合条件的每个会员账户 → 系统将优惠券状态设为"已发放"，记录发放时间戳
- Success End: 优惠券状态为"已发放"；会员账户下优惠券列表状态为"已更新"
- Exception Paths:
  - IF 店长配置的优惠券有效期早于当前日期：系统将创建状态设为"有效期无效"，向店长展示"有效期不得早于今日，请重新设置"，店长可修改有效期后重新保存。
  - IF 按等级筛选后符合条件的会员数为0：系统将发放状态设为"无匹配会员"，向店长展示"当前无符合该等级的会员，请调整发放对象"，店长可修改等级范围后重新发放。

### Scenario: 店长查看数据报表
- Trigger: 店长在数据报表模块选择查询维度和时间范围后提交查询时触发
- Flow:
  1. 店长在数据报表选择查询维度（会员活跃度/营收分析/复购率）和时间范围并提交 → 系统在消费记录中聚合对应时间段的消费数据，将报表状态设为"生成中"
  2. 系统在数据报表将聚合结果渲染展示（活跃会员数/营收金额/复购率/客单价等） → 系统将报表状态设为"已生成"
- Success End: 报表查询结果状态为"已展示"
- Exception Paths:
  - IF 所选时间范围内消费记录为空：系统将报表状态设为"无数据"，向店长展示"所选时间段内暂无消费数据"，店长可调整时间范围后重新查询。
  - IF 报表聚合计算超时：系统将报表状态设为"加载失败"，向店长展示"数据加载超时，请稍后重试"，店长可重新发起查询。

## Module Connections

| 上游模块 | 触发操作 | 下游模块 | 自动反应 |
|---------|---------|---------|---------|
| 消费记录 | 店员提交消费金额，消费明细写入状态为"已记录" | 积分管理 | 按消费金额（1元=1分）累加该会员积分余额 |
| 消费记录 | 店员提交消费金额，消费明细写入状态为"已记录" | 会员管理 | 将消费金额累加至年度累计消费额 |
| 会员管理 | 系统累加年度累计消费额后触发阈值检测 | 会员管理（等级字段） | 根据检测结果将会员等级更新为对应等级（银卡/金卡/钻卡） |
| 优惠券管理 | 店长点击确认发放 | 会员管理 | 查询符合等级范围的会员列表并返回匹配结果 |
| 优惠券管理 | 系统完成优惠券与会员账户绑定，状态设为"已发放" | 会员管理 | 会员账户下优惠券列表追加新券记录，状态为"已发放" |
| 消费记录 | 店员录入含优惠券抵扣信息的消费明细并提交 | 优惠券管理 | 将所用优惠券状态设为"已使用"，记录核销时间戳 |
| 消费记录 | 店员录入含优惠券抵扣信息的消费明细并提交 | 积分管理 | 按实付金额（扣减优惠券后，1元=1分）累加该会员积分余额 |
| 消费记录 | 店员录入含优惠券抵扣信息的消费明细并提交 | 会员管理 | 将实付金额累加至年度累计消费额 |
| 消费记录 | 店长提交报表查询请求 | 数据报表 | 聚合对应时间段消费明细，返回活跃度、营收、复购率统计数据并渲染展示 |

## Trigger Rules

| 编号 | 触发条件 | 触发动作 | 通知对象 |
|------|---------|---------|---------|
| T1 | 每年 12 月 31 日 | 系统在积分管理将所有会员积分余额清零，写入积分变更记录（类型：年度清零） | 会员 |
| T2 | 每月定时检测，发现某会员在消费记录中连续 6 个月无消费记录 | 系统在会员管理将该会员等级降低一级（钻卡→金卡、金卡→银卡、银卡维持不变），写入等级变更记录（类型：连续无消费降级） | 店长 |
