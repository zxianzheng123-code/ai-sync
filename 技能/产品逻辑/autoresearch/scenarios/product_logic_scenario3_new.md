# Product Logic

## Instructions For Design & Coding AI
- 本文档定义系统运转逻辑：场景、流程、联动、触发规则。
- 页面设计必须覆盖本文档中的所有场景和联动点。
- 编码时的数据流和事件处理必须与本文档的触发规则对齐。
- 发现逻辑矛盾或缺口时，以本文档为准，向上游反馈。
- 预约提交时系统须校验疫苗记录是否在 30 天内，不符合条件的宠物禁止提交预约，系统须在提交入口屏蔽确认按钮并展示疫苗过期原因。
- 节假日预约必须提前 72 小时，普通日期必须提前 24 小时，不满足时间条件的请求禁止写入预约记录。
- 取消预约时系统须自动计算距离入住时间，不得允许退款金额超出实际已支付定金。
- 寄养期间每天至少上传 1 张宠物照片，寄养员不得跳过当日照片上传直接提交日志记录。

## Business Scenarios

### Scenario: 宠物主人提交寄养预约
- Trigger: 宠物主人 已登录，在预约管理模块选择寄养日期并点击"提交预约"时触发
- Flow:
  1. 宠物主人 在 预约管理 选择寄养开始和结束日期 → 系统计算距当前时间间隔，校验是否满足提前 24 小时（节假日 72 小时）条件
  2. 宠物主人 在 预约管理 选择房间类型 → 系统展示对应房间的可用数量
  3. 宠物主人 在 宠物档案 选择要寄养的宠物 → 系统读取该宠物最新疫苗接种记录，校验是否在 30 天内
  4. 宠物主人 在 预约管理 点击"提交预约" → 系统将预约状态写为"待确认"，写入预约 ID、宠物 ID、房间类型、寄养日期段
  5. 系统 在 通知管理 预约状态写为"待确认"后 → 向宠物主人推送预约确认通知，携带预约 ID 和寄养日期
- Success End: 预约状态为"待确认"，宠物主人收到预约确认通知，预约记录写入系统
- Exception Paths:
  - IF 提交时距入住时间不足 24 小时（普通日）或不足 72 小时（节假日）：系统拒绝写入预约，展示"预约需提前至少 24 小时（节假日 72 小时），请重新选择日期"，预约状态不变
  - IF 宠物疫苗记录不在 30 天内：系统屏蔽"提交预约"按钮，展示"[宠物名] 疫苗记录已过期，请先更新健康档案后再提交预约"，预约不写入
  - IF 所选房间类型当日无空余：系统展示"所选日期该房间类型已满，请选择其他房间类型或日期"，预约不写入

### Scenario: 宠物主人取消预约并申请退款
- Trigger: 宠物主人 已登录，在预约管理模块找到"待确认"或"已确认"状态的预约，点击"取消预约"时触发
- Flow:
  1. 宠物主人 在 预约管理 点击"取消预约" → 系统计算当前时间距寄养入住时间的间隔
  2. 系统 在 支付管理 根据间隔判断退款比例 → 提前 48 小时以上全额退还定金；不足 48 小时退还定金的 50%
  3. 宠物主人 在 预约管理 确认取消 → 系统将预约状态变更为"已取消"，向支付管理写入退款金额和退款原因
  4. 系统 在 支付管理 退款记录写入后 → 执行退款至宠物主人原支付渠道，记录退款完成时间戳
  5. 系统 在 通知管理 退款执行后 → 向宠物主人推送"预约已取消，退款 [X] 元已原路返回"通知
- Success End: 预约状态为"已取消"，退款已执行，宠物主人收到取消和退款通知
- Exception Paths:
  - IF 宠物主人取消时距入住不足 48 小时：系统展示"当前距入住不足 48 小时，取消将扣除定金的 50%，退还 [X] 元，确认继续？"，宠物主人确认后方执行取消
  - IF 预约状态为"寄养中"（宠物已入住）：系统拒绝取消操作，展示"宠物已入住，不支持在线取消，请联系店长处理"，预约状态不变

### Scenario: 寄养员记录当日寄养日志
- Trigger: 寄养员 已登录，在寄养日志模块为某在住宠物新建当日记录时触发
- Flow:
  1. 寄养员 在 寄养日志 选择在住宠物并新建当日日志 → 系统创建日志草稿，关联宠物 ID 和当日日期
  2. 寄养员 在 寄养日志 填写喂食记录（时间、食物类型、食量） → 系统将喂食数据写入日志草稿
  3. 寄养员 在 寄养日志 填写活动记录和健康观察 → 系统将活动和健康数据写入日志草稿
  4. 寄养员 在 寄养日志 上传至少 1 张宠物照片 → 系统存储照片并标记"已上传"，照片与当日日志关联
  5. 寄养员 在 寄养日志 点击"提交日志" → 系统将日志状态变更为"已提交"，记录提交时间戳
  6. 系统 在 通知管理 日志状态变更为"已提交"后 → 向宠物主人推送每日寄养报告，包含喂食、活动摘要和照片
- Success End: 日志状态为"已提交"，宠物主人收到当日寄养报告推送
- Exception Paths:
  - IF 寄养员未上传任何照片直接点击"提交日志"：系统拒绝提交，屏蔽"提交"按钮，展示"每日至少上传 1 张宠物照片才能提交日志"，日志保持草稿状态
  - IF 寄养员提交的健康观察标记"异常"：系统将该日志标记"健康预警"，立即向宠物主人推送"[宠物名] 今日健康观察异常，请查看日志详情"紧急通知，同时通知店长

### Scenario: 店长管理房间排期
- Trigger: 店长 已登录，在预约管理模块查看房间占用情况，调整房间类型或排期时触发
- Flow:
  1. 店长 在 预约管理 查看当前所有预约的房间占用日历 → 系统展示各房间类型的占用状态、入住和离开日期
  2. 店长 在 预约管理 将某房间类型标记为"停用"（维修或特殊安排） → 系统将该房间类型可用数量更新为 0，关联该房间的新预约入口自动关闭
  3. 店长 在 预约管理 确认停用操作 → 系统检查该房间类型是否有未来已确认的预约
  4. 系统 在 通知管理 检测到停用房间有已确认预约 → 向受影响宠物主人推送"您的预约房间 [类型] 因调整不可用，请联系店长重新安排"通知
- Success End: 房间类型状态已更新，受影响宠物主人已收到通知
- Exception Paths:
  - IF 店长停用房间时该房间类型有宠物当前在住：系统拒绝停用操作，展示"该房间类型当前有 [N] 只宠物在住，无法停用，请等待当前寄养结束"
  - IF 店长恢复已停用房间为可用：系统将该房间类型可用数量恢复至正常值，向曾因该类型无房被拒绝的待确认预约主人推送"[房间类型] 已恢复可用，您可重新提交预约"通知

### Scenario: 宠物主人查阅宠物档案并更新疫苗记录
- Trigger: 宠物主人 已登录，在宠物档案模块发现疫苗记录即将过期或已过期，点击"更新疫苗记录"时触发
- Flow:
  1. 宠物主人 在 宠物档案 查看宠物的疫苗接种记录列表 → 系统展示每条疫苗的接种日期和距今天数
  2. 宠物主人 在 宠物档案 点击"更新疫苗记录"并填写新接种日期和疫苗类型 → 系统校验接种日期不得晚于今天
  3. 宠物主人 在 宠物档案 提交新疫苗记录 → 系统将新记录写入健康档案，标记最新疫苗接种日期
  4. 系统 在 宠物档案 疫苗记录更新后 → 重新校验该宠物关联的所有"因疫苗过期被拒绝"的待提交预约，向宠物主人推送"档案已更新，可重新提交预约"提示
- Success End: 新疫苗记录已写入宠物档案，档案中最新疫苗接种日期在 30 天内，宠物可提交预约
- Exception Paths:
  - IF 宠物主人填写的疫苗接种日期晚于今天（未来日期）：系统拒绝写入，展示"接种日期不得填写未来日期，请填写实际接种日期"
  - IF 宠物主人无该宠物的编辑权限（非本账号宠物）：系统拒绝操作，展示"您没有权限编辑该宠物档案"，档案数据不变

## Module Connections

| 上游模块 | 触发操作 | 下游模块 | 自动反应 |
|---------|---------|---------|---------|
| 预约管理 | 预约状态写为"待确认" | 通知管理 | 向宠物主人推送预约确认通知，携带预约 ID、寄养日期段和房间类型 |
| 预约管理 | 预约状态变更为"已取消" | 支付管理 | 写入退款记录，包含退款金额（全额或 50% 定金）、退款原因、原支付渠道信息 |
| 支付管理 | 退款记录写入并执行完成 | 通知管理 | 向宠物主人推送退款完成通知，携带退款金额和预约 ID |
| 寄养日志 | 日志状态变更为"已提交" | 通知管理 | 向宠物主人推送每日寄养报告，携带喂食摘要、活动摘要和当日照片链接 |
| 宠物档案 | 疫苗记录更新且接种日期在 30 天内 | 预约管理 | 解除该宠物因疫苗过期导致的预约提交限制，恢复提交按钮可用状态 |
| 预约管理 | 店长将房间类型标记为"停用" | 通知管理 | 向该房间类型所有未来已确认预约的宠物主人推送房间调整通知 |

## Trigger Rules

| 编号 | 触发条件 | 触发动作 | 通知对象 |
|------|---------|---------|---------|
| T1 | 宠物主人提交预约时距入住不足 24 小时（普通日）或不足 72 小时（节假日） | 拒绝写入预约，展示时间不足提示，预约状态不变 | 宠物主人 |
| T2 | 宠物主人提交预约时宠物疫苗记录超出 30 天 | 屏蔽"提交预约"按钮，展示疫苗过期原因 | 宠物主人 |
| T3 | 宠物主人取消预约且距入住时间不足 48 小时 | 扣除定金的 50%，退还剩余 50%，展示确认提示 | 宠物主人 |
| T4 | 寄养员提交日志时未上传任何照片 | 拒绝提交，日志保持草稿，展示"至少上传 1 张照片"提示 | 寄养员 |
| T5 | 寄养日志健康观察标记为"异常" | 立即推送健康预警通知，日志标记"健康预警" | 宠物主人、店长 |
| T6 | 寄养日志状态变更为"已提交" | 向宠物主人推送每日寄养报告，含喂食、活动摘要和照片 | 宠物主人 |
| T7 | 预约状态变更为"已取消"且已支付定金 | 按规则计算退款金额，写入退款记录，执行退款至原支付渠道 | 宠物主人 |
