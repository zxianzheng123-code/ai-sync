# Product Logic

## Instructions For Design & Coding AI
- 本文档定义系统运转逻辑：场景、流程、联动、触发规则。
- 页面设计必须覆盖本文档中的所有场景和联动点。
- 编码时的数据流和事件处理必须与本文档的触发规则对齐。
- 发现逻辑矛盾或缺口时，以本文档为准，向上游反馈。
- 预约提交前系统必须强制校验宠物档案中的疫苗接种记录是否在 30 天内有效，不得绕过校验直接入住。
- 取消预约时系统不得由店员手动决定退款比例，退款比例由系统依据取消时间自动判断（提前 48 小时全额退，否则扣 50%）。
- 寄养期间每日至少上传 1 张照片，系统禁止在当日照片未上传前将当日日志标记为"已完成"。
- 节假日预约须提前 72 小时，普通预约须提前 24 小时，系统不允许接受不满足提前时限的预约提交。

## Business Scenarios

### Scenario: 宠物主人提交寄养预约
- Trigger: 宠物主人在预约管理模块选择寄养日期和房间类型并提交预约时触发
- Flow:
  1. 宠物主人 在 预约管理 选择寄养起止日期和房间类型 → 系统检查所选日期距当前时间是否满足提前时限（普通日 24 小时、节假日 72 小时）
  2. 系统 在 宠物档案 查询该宠物最近一次疫苗接种记录 → 系统判断接种日期是否在 30 天内
  3. 宠物主人 在 预约管理 确认预约信息并提交 → 系统生成预约单，状态标记为"待付定金"
  4. 宠物主人 在 支付管理 完成定金支付 → 系统将预约状态更新为"已确认"
  5. 系统 在 通知管理 向宠物主人发送预约确认通知 → 通知包含预约编号、寄养日期、房间类型
- Success End: 预约单状态为"已确认"，定金支付记录已写入，预约确认通知已发出
- Exception Paths:
  - IF 所选日期不满足提前时限（普通日不足 24 小时 / 节假日不足 72 小时）：系统在预约管理模块拦截提交，提示"预约须至少提前 [24/72] 小时，请重新选择日期"，预约单不生成
  - IF 宠物疫苗接种记录不在 30 天内或无记录：系统在预约管理模块拦截提交，提示"宠物档案中无 30 天内有效疫苗记录，请先更新宠物档案后再预约"，预约单不生成
  - IF 定金支付超时未完成：系统将预约单状态标记为"已失效"，释放该房间时段，通知管理模块向宠物主人发送"预约已失效，如需重新预约请重新提交"提示

### Scenario: 宠物主人取消预约
- Trigger: 宠物主人在预约管理模块发起取消预约操作时触发
- Flow:
  1. 宠物主人 在 预约管理 选择待取消的预约单并发起取消 → 系统读取该预约单的寄养起始日期，计算与当前时间的差值
  2. 系统 在 支付管理 根据时间差判断退款比例（提前 48 小时及以上全额退定金，不足 48 小时退 50% 定金）→ 系统生成退款申请，写入退款金额和扣款比例
  3. 系统 在 预约管理 将预约单状态更新为"已取消" → 释放对应房间时段
  4. 系统 在 支付管理 执行退款至宠物主人原支付渠道 → 退款记录写入支付流水
  5. 系统 在 通知管理 向宠物主人发送取消确认通知 → 通知包含退款金额和预计到账时间
- Success End: 预约单状态为"已取消"，退款记录已写入，对应房间时段已释放，取消通知已发出
- Exception Paths:
  - IF 预约单状态已为"已取消"或"已失效"：系统提示"该预约已处于取消状态，无需重复操作"，不产生新的退款记录
  - IF 宠物已办理入住（寄养已开始）：系统拦截取消操作，提示"寄养已开始，无法通过预约管理取消，请联系店长处理"，预约单状态不变

### Scenario: 寄养员记录每日寄养日志
- Trigger: 寄养员在寄养日志模块为当日在住宠物录入日常记录时触发
- Flow:
  1. 寄养员 在 寄养日志 选择对应宠物并新建当日日志 → 系统展示该宠物档案的特殊需求备注供参考
  2. 寄养员 在 寄养日志 填写喂食记录、活动记录、健康观察 → 系统暂存日志草稿，状态为"未完成"
  3. 寄养员 在 寄养日志 上传当日宠物照片（至少 1 张）→ 系统记录上传时间和照片数量
  4. 寄养员 在 寄养日志 提交当日日志 → 系统将日志状态更新为"已完成"
  5. 系统 在 通知管理 向宠物主人推送每日寄养报告 → 报告包含喂食记录、健康观察摘要和当日照片
- Success End: 当日寄养日志状态为"已完成"，照片已上传，每日报告已推送给宠物主人
- Exception Paths:
  - IF 寄养员尝试在未上传照片的情况下提交日志：系统拦截提交，提示"当日照片未上传，须至少上传 1 张照片后才能提交日志"，日志状态保持"未完成"
  - IF 当日日志未在寄养结束前完成（截止检查时仍为"未完成"）：系统在寄养日志模块将该日志标记为"逾期未提交"，同时在通知管理模块向店长发送告警通知，说明哪只宠物当日日志缺失

### Scenario: 店长管理房间排期
- Trigger: 店长在预约管理模块查看房间占用情况并调整排期时触发
- Flow:
  1. 店长 在 预约管理 查看当前房间占用情况和排期日历 → 系统展示各房间类型的预约状态和空余时段
  2. 店长 在 预约管理 确认某时段某房间是否可接受新预约 → 系统展示该时段内已确认的预约单列表
  3. 店长 在 预约管理 设置特定日期为节假日 → 系统将该日期提前时限规则更新为 72 小时
- Success End: 排期日历数据已更新，节假日标记已生效，后续提交预约时系统将按更新后的规则校验
- Exception Paths:
  - IF 店长尝试将已有"已确认"预约单的时段标记为不可预约：系统提示"该时段已有 [N] 个确认预约，无法直接关闭，请先联系相关宠物主人处理"，不修改现有预约单状态
  - IF 店长设置节假日的日期距当前不足 72 小时且该日期已有普通预约提交：系统提示"该日期已有预约按普通规则受理，节假日标记对已提交预约不追溯生效"，仅对新提交预约生效

### Scenario: 宠物主人接宠离店
- Trigger: 宠物主人到店接宠，寄养员在预约管理模块办理离店手续时触发
- Flow:
  1. 寄养员 在 预约管理 确认宠物主人身份并核对预约单 → 系统展示该预约单的寄养信息和尾款状态
  2. 宠物主人 在 支付管理 完成尾款支付 → 系统将支付状态标记为"尾款已付"
  3. 寄养员 在 预约管理 办理宠物离店 → 系统将预约单状态更新为"已完成"，房间状态释放
  4. 系统 在 通知管理 向宠物主人发送离店确认通知 → 通知包含本次寄养费用明细和寄养日志链接
- Success End: 预约单状态为"已完成"，尾款已结清，房间已释放，离店通知已发出
- Exception Paths:
  - IF 尾款支付失败或宠物主人拒付：系统在支付管理模块保留尾款待支付记录，在预约管理模块将预约单标记为"待结算"，系统在通知管理模块向店长发送"尾款未结清"告警通知，不执行离店操作
  - IF 寄养期间存在未完成的日志（日志状态仍为"未完成"或"逾期未提交"）：系统在办理离店时提示"存在 [N] 条未完成日志记录"，店长需确认后方可继续离店流程，不自动强制阻止

## Module Connections

| 上游模块 | 触发操作 | 下游模块 | 自动反应 |
|---------|---------|---------|---------|
| 宠物档案 | 疫苗接种记录被读取校验 | 预约管理 | 返回校验结果（有效/无效），决定预约单是否允许生成 |
| 预约管理 | 预约单状态更新为"已确认" | 通知管理 | 生成预约确认通知，写入宠物主人账户，推送通知内容含预约编号和寄养日期 |
| 支付管理 | 定金支付成功写入 | 预约管理 | 将预约单状态从"待付定金"更新为"已确认" |
| 预约管理 | 宠物主人发起取消，系统计算退款比例 | 支付管理 | 生成退款记录，写入退款金额（全额或 50%），执行退款至原支付渠道 |
| 支付管理 | 退款执行完毕 | 通知管理 | 向宠物主人推送取消确认通知，包含退款金额和预计到账时间 |
| 寄养日志 | 当日日志状态更新为"已完成" | 通知管理 | 生成每日寄养报告，推送给对应宠物主人，包含喂食记录、健康观察和当日照片 |
| 宠物档案 | 特殊需求备注被读取 | 寄养日志 | 在新建日志时展示该宠物的特殊需求备注供寄养员参考 |
| 支付管理 | 尾款支付成功写入 | 预约管理 | 将预约单支付状态更新为"尾款已付"，允许执行离店操作 |
| 预约管理 | 预约单状态更新为"已完成" | 通知管理 | 生成离店确认通知，推送给宠物主人，包含费用明细和寄养日志链接 |

## Trigger Rules

| 编号 | 触发条件 | 触发动作 | 通知对象 |
|------|---------|---------|---------|
| T1 | 预约提交时，所选日期距当前不足 24 小时（普通日）或不足 72 小时（节假日） | 预约管理模块拦截提交，预约单不生成 | 宠物主人（预约管理模块提示具体所需提前时限） |
| T2 | 预约提交时，宠物疫苗接种记录不在 30 天内或无记录 | 预约管理模块拦截提交，预约单不生成 | 宠物主人（预约管理模块提示需更新宠物档案） |
| T3 | 定金支付超时未完成 | 预约单状态标记为"已失效"，对应房间时段释放 | 宠物主人（通知管理模块发送预约失效提醒） |
| T4 | 寄养日志当日截止时仍为"未完成"状态 | 寄养日志模块将日志标记为"逾期未提交" | 店长（通知管理模块发送日志缺失告警，注明宠物信息） |
| T5 | 离店办理时尾款支付失败 | 支付管理模块保留待支付记录，预约单标记为"待结算" | 店长（通知管理模块发送"尾款未结清"告警通知） |
