# Product Logic

## Instructions For Design & Coding AI
- 本文档定义系统运转逻辑：场景、流程、联动、触发规则。
- 页面设计必须覆盖本文档中的所有场景和联动点。
- 编码时的数据流和事件处理必须与本文档的触发规则对齐。
- 发现逻辑矛盾或缺口时，以本文档为准，向上游反馈。
- 校服按学校定制，系统不得允许跨学校混发：下单时必须校验商品归属学校与客户档案学校一致，不一致则拒绝下单。
- 大单（500 件及以上）禁止使用在线支付通道，必须走对公转账；小单（500 件以下）禁止使用对公转账通道。
- 退货申请须在发货后 7 天内提交，且校服须为未穿着状态，否则系统不得受理。
- 库存低于安全线时，系统必须自动提醒校服零售商（老板）补货，不得静默忽略。

## Business Scenarios

### Scenario: 学校采购员下单
- Trigger: 学校采购员在客户档案已建立、商品目录已上架的条件下，进入订单管理发起新订单时触发
- Flow:
  1. 学校采购员 在 客户管理 确认本校客户档案已存在 → 系统展示该学校的历史合作记录和联系人信息
  2. 学校采购员 在 商品管理 选择本校对应款式和尺码 → 系统校验商品归属学校与当前客户档案学校一致，展示各尺码当前可用库存数量
  3. 学校采购员 在 订单管理 填写下单数量并提交订单 → 系统生成待支付订单，记录订单所属学校、款式、尺码、数量
  4. 系统 在 支付结算 判断订单数量 → 500 件及以上路由至对公转账通道，500 件以下路由至在线支付通道
  5. 学校采购员 在 支付结算 完成对应支付操作 → 系统将订单状态更新为已支付，账单金额和支付方式写入结算记录
  6. 系统 在 库存管理 锁定对应款式尺码库存 → 该 SKU 可用库存减少，锁定库存增加，锁定记录写入库存日志
- Success End: 订单状态为已支付，库存已锁定，账单已写入结算记录
- Exception Paths:
  - IF 学校采购员选择的商品归属学校与其客户档案学校不一致: 系统拒绝将该商品加入订单，商品管理页面显示"该款式不属于贵校，不可下单"，采购员须重新选择本校款式
  - IF 订单数量达到 500 件但采购员选择在线支付通道: 系统拦截支付请求，订单状态保持为待支付，支付结算页面显示"500 件及以上订单须走对公转账"，引导采购员切换通道
  - IF 下单数量超过当前可用库存: 系统拒绝提交订单，订单管理页面显示当前可用库存数量，采购员可修改数量或联系校服零售商（老板）

### Scenario: 校服零售商（老板）录入新款式
- Trigger: 校服零售商（老板）在新增合作学校或学校更换款式时，进入商品管理发起录入操作时触发
- Flow:
  1. 校服零售商（老板） 在 客户管理 确认目标学校客户档案已建立 → 系统展示该学校档案，学校标识可绑定至新款式
  2. 校服零售商（老板） 在 商品管理 填写款式名称、绑定学校、价格、尺码表并提交 → 系统将款式与指定学校绑定，生成商品记录，状态为待上架
  3. 校服零售商（老板） 在 商品管理 确认信息无误后将款式状态改为上架 → 系统将该款式设为可下单状态，仅对绑定学校的采购员可见
  4. 校服零售商（老板） 在 库存管理 录入该款式各尺码的初始库存数量和安全线数值 → 系统写入各 SKU 当前库存量和安全库存阈值
- Success End: 新款式已上架，绑定学校正确，各尺码初始库存和安全线已录入
- Exception Paths:
  - IF 目标学校客户档案不存在: 系统在商品管理中阻止绑定学校操作，提示"请先在客户管理中为该学校建立档案"，老板须先完成建档再返回录入
  - IF 尺码表为空或价格未填写: 系统拒绝保存，商品管理页面标红缺失字段，老板须补全后重新提交

### Scenario: 仓库管理员处理出库发货
- Trigger: 仓库管理员在订单状态变更为已支付后，收到出库任务进入库存管理操作时触发
- Flow:
  1. 仓库管理员 在 订单管理 查看待出库订单列表 → 系统展示所有已支付待发货订单，含学校、款式、尺码、数量信息
  2. 仓库管理员 在 库存管理 核对实物与订单中的款式、尺码、学校信息 → 系统显示该订单对应 SKU 的锁定库存数量
  3. 仓库管理员 在 库存管理 确认出库并提交出库记录 → 系统将该 SKU 锁定库存清零，实际库存扣减对应数量，出库时间写入库存日志
  4. 仓库管理员 在 订单管理 将订单状态更新为已发货 → 系统记录发货时间戳，订单状态变更为已发货，7 天退货窗口开始计时
  5. 系统 在 库存管理 检查出库后该 SKU 实际库存是否低于安全线 → 若低于安全线则写入补货提醒条目并通知校服零售商（老板）
- Success End: 订单状态为已发货，实际库存已扣减，出库记录已写入，发货时间戳已记录
- Exception Paths:
  - IF 实物款式或尺码与订单记录不一致: 仓库管理员在库存管理中停止出库操作，在订单管理中将订单标记为发货异常，通知校服零售商（老板）处理，订单状态保持为待发货
  - IF 实际库存数量与系统锁定数量不符: 系统拒绝出库提交，库存管理页面显示系统数量与盘点差异，仓库管理员须先在库存管理中完成盘点修正再提交出库

### Scenario: 学校采购员申请退货
- Trigger: 学校采购员在收到货物后发现问题，在发货后 7 天内进入订单管理发起退货申请时触发
- Flow:
  1. 学校采购员 在 订单管理 找到已发货订单并点击申请退货 → 系统校验当前时间距发货时间戳是否在 7 天以内
  2. 学校采购员 在 订单管理 填写退货原因并勾选"校服未穿着"确认项后提交 → 系统生成退货申请单，订单状态变更为退货待审核
  3. 校服零售商（老板） 在 订单管理 审核退货申请，确认退货条件后批准 → 系统将订单状态变更为退货已批准，通知学校采购员可寄回实物
  4. 仓库管理员 在 库存管理 确认退回实物已到货并验收完成 → 系统将对应 SKU 库存加回，退货入库记录写入库存日志
  5. 系统 在 支付结算 生成退款账单 → 退款金额写入结算记录，退款状态标记为待处理
- Success End: 退货入库完成，库存已恢复，退款账单已生成，订单状态为退货完成
- Exception Paths:
  - IF 申请时间距发货时间戳超过 7 天: 系统拒绝受理退货申请，订单管理页面显示"退货申请须在发货后 7 天内提交，当前已超期"，订单状态不变
  - IF 学校采购员未勾选"校服未穿着"确认项: 系统拒绝提交退货申请，订单管理页面提示"退货须为未穿着校服，请确认后重新提交"
  - IF 校服零售商（老板）审核后驳回申请: 系统将订单状态恢复为已发货，向学校采购员展示驳回原因，采购员可重新申请或放弃

### Scenario: 校服零售商（老板）查看结算与对账
- Trigger: 校服零售商（老板）需要核对当期账单或确认收款状态时，进入支付结算模块时触发
- Flow:
  1. 校服零售商（老板） 在 支付结算 查看账单列表 → 系统展示当期所有已支付订单账单和退款记录
  2. 校服零售商（老板） 在 支付结算 点击单条账单查看明细 → 系统展示该账单对应的订单编号、学校名称、金额、支付方式、支付时间
  3. 校服零售商（老板） 在 支付结算 执行对账操作确认到账金额与账单金额一致 → 系统将账单状态更新为已对账，记录对账时间
- Success End: 账单状态为已对账，对账时间已记录
- Exception Paths:
  - IF 到账金额与账单金额不一致: 校服零售商（老板）在支付结算中将该账单标记为对账异常，系统生成异常记录，账单状态保持为待对账，等待人工跟进
  - IF 对公转账订单长时间未到账: 校服零售商（老板）在支付结算中将该账单标记为转账跟进中，系统记录跟进状态，账单状态保持为待对账

## Module Connections

| 上游模块 | 触发操作 | 下游模块 | 自动反应 |
|---------|---------|---------|---------|
| 客户管理 | 新学校档案建立 | 商品管理 | 新学校标识写入可绑定学校列表，老板录入新款式时可选该学校 |
| 商品管理 | 新款式上架并绑定学校 | 订单管理 | 该款式加入对应学校的可下单商品目录，学校采购员可见 |
| 支付结算 | 在线支付完成 | 订单管理 | 订单状态从待支付更新为已支付，支付金额和时间写入账单记录 |
| 支付结算 | 对公转账确认到账 | 订单管理 | 订单状态从待支付更新为已支付，到账金额和时间写入账单记录 |
| 订单管理 | 订单状态变更为已支付 | 库存管理 | 对应款式尺码 SKU 的可用库存减少、锁定库存增加，锁定记录写入库存日志 |
| 库存管理 | 出库确认提交 | 订单管理 | 订单出库完成标记写入，订单可更新为已发货状态 |
| 订单管理 | 订单状态变更为已发货 | 库存管理 | 该 SKU 锁定库存清零，实际库存扣减，发货时间戳写入出库日志 |
| 库存管理 | 出库后 SKU 实际库存低于安全线 | 订单管理 | 补货提醒条目写入待办，提示校服零售商（老板）关注库存状态 |
| 订单管理 | 退货申请审核通过且实物验收完成 | 库存管理 | 退回 SKU 库存加回，退货入库记录写入库存日志 |
| 订单管理 | 退货入库确认完成 | 支付结算 | 退款金额写入结算记录，退款账单生成并标记为待处理 |

## Trigger Rules

| 编号 | 触发条件 | 触发动作 | 通知对象 |
|------|---------|---------|---------|
| T1 | 出库操作完成后，库存管理中某 SKU 实际库存低于预设安全线 | 系统在库存管理中生成补货提醒条目，标注 SKU 名称、当前库存量、安全线数值 | 校服零售商（老板） |
| T2 | 订单状态变更为已发货后，系统计算当前时间距发货时间戳超过 7 天 | 系统关闭该订单的退货申请入口，订单状态标注退货窗口已关闭 | 学校采购员 |
| T3 | 学校采购员提交退货申请时，系统校验当前时间距发货时间戳超过 7 天 | 系统拒绝受理，退货申请单不写入，订单管理页面显示超期提示 | 学校采购员 |
| T4 | 下单时系统检测商品归属学校与客户档案学校不一致 | 系统阻止该商品加入订单，商品管理页面显示学校不一致错误提示 | 学校采购员 |
| T5 | 订单数量达到 500 件，采购员选择在线支付通道 | 系统拦截支付请求，支付结算页面显示"500 件及以上须走对公转账"提示，订单状态保持为待支付 | 学校采购员 |
