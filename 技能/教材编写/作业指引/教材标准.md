# 教材标准

> 定义教材编写工作流在产出「教材」时必须遵守的规则。本文件只管"如何让别人读懂、学会、传承"；不管把事情压成 SOP，也不负责直接做判断。

## 一、教材定义

「教材」指的是：

- 课本
- 训练资料
- 系统化经验文档
- 面向新人或团队的教学材料

核心特征：读完之后能建立完整认知，知其然也知其所以然。

## 二、进入条件

进入「教材」前，至少满足：

1. 题目已说清
2. 核心事实和结论已经稳定
3. 你真正想要的是"别人能学会"

不满足时的回退：

- 题目还乱 → `问题拆解师`
- 关键事实或判断未定 → `调研判断师`
- 你真正要的是 SOP / 规则 / 流程 → `方法提炼师`

## 三、教材结构

一个合格的教材文档至少包含：

1. **目标读者** — 这份教材写给谁
2. **学习目标** — 读完之后能做到什么
3. **背景和目标** — 为什么要学这个、解决什么问题
4. **核心原理** — 底层逻辑是什么
5. **具体做法** — 实际怎么操作
6. **案例与常见坑** — 真实场景中的正面和反面例子
7. **学完后的检查点** — 怎么确认自己真的学会了

## 四、教材 vs 方法 判断清单

≥ 2 个答"是"就是教材，否则考虑方法：

1. 读者需要理解"为什么这样做"才能正确执行吗？
2. 文档核心是"背景 + 原理 + 案例 + 常见坑"而不是"步骤 + 触发 + 验收"？
3. 目标读者是"第一次接触、需要建立完整认知的人"而不是"已经懂了、只需照做的人"？

## 五、边界提醒

- `教材` 重"让人学会" — 需要讲清背景和原理
- `教材` 不是更长的 `方法` — 如果主要是步骤和触发条件，说明它正在滑向方法
- `教材` 不负责直接做推荐 — 推荐在上游已经完成
- 如果文档主要是步骤、触发条件和例外处理，应转给 `方法提炼师`
