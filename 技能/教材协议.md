# 教材协议

> 定义所有技能在产出 `教材` 这类最终成果时必须遵守的规则。本协议只管"如何让别人读懂、学会、传承"；不管把事情压成 SOP，也不负责直接做判断。

## 一、教材定义

`教材` 指的是：

- 课本
- 训练资料
- 系统化经验文档
- 面向新人或团队的教学材料

## 二、进入条件

进入 `教材` 前，至少满足：

1. 题目已说清
2. 核心事实和结论已经稳定
3. 你真正想要的是"别人能学会"

不满足时的回退：

- 题目还乱 → `问题拆解师`
- 关键事实或判断未定 → `调研判断师`
- 你真正要的是 SOP / 规则 / 流程 → `方法提炼师`

## 三、教材结构

一个合格的教材文档至少包含：

1. 目标读者
2. 学习目标
3. 背景和目标
4. 核心原理
5. 具体做法
6. 案例与常见坑
7. 学完后的检查点

## 四、边界提醒

- `教材` 重"让人学会"
- `教材` 不是更长的 `方法`
- `教材` 不负责直接做推荐

如果文档主要是步骤、触发条件和例外处理，说明它正在滑向 `方法`。
