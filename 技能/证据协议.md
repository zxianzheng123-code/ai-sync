# 证据协议

> 定义所有技能在需要信息检索、资料收集、来源验证时必须遵守的规则。本协议只管"怎么找证据、筛证据、验证据、标证据"；不管题目是否说清，也不管最终如何给推荐。

## 一、适用边界

以下场景进入证据协议：

- 需要网页搜索、文档查找、资料比对
- 需要确认事实、数据、价格、版本、来源
- 需要判断手上信息是否足够可信

以下场景不归本协议管：

- 当前到底要查什么 → 归 `成题协议`
- 证据到手后如何下结论 → 归 `判断协议`

## 二、检索启动条件

只有在以下条件同时满足时，才开始检索：

1. 已经知道要找什么
2. 已经知道大概找给谁用、用来做什么
3. 已经知道检索范围至少要覆盖哪些维度

缺任何一项，先回 `成题协议` 补题。

## 三、检索输入

每次检索前，至少明确这 4 项：

1. **检索对象**：要找什么事实/数据/材料
2. **检索目的**：找回来是为了回答什么问题
3. **优先维度**：最关键的 2-4 个比较或验证维度
4. **时效要求**：有没有最近、最新、当次有效的要求

## 四、检索策略

### 4.1 先找哪里

默认优先顺序：

1. 你亲口提供的信息
2. 官方文档 / 官方数据 / 一手来源
3. 可追溯的第三方资料
4. 社区经验或间接来源

### 4.2 怎么找

检索时必须显式写出策略，让你知道我在干什么：

- 第一轮：先用最直白的词搜（就像你自己搜索一样），定位主对象
- 第二轮：换同义词或行业词补遗漏，因为同一个东西不同人叫法不同
- 第三轮：按关键维度分开搜，确保每个重要角度都有覆盖

### 4.3 什么时候停

满足以下任一可停：

- 关键维度已被硬证据或有参考价值的来源覆盖
- 同类结果开始重复
- 发现该维度根本没有可验证来源

## 五、来源分级

| 级别 | 说明 | 处理方式 |
| --- | --- | --- |
| `硬证据` | 你亲口说的、官方文档、一手数据 | 可直接作为核心依据 |
| `有参考价值` | 可追溯第三方资料、行业报告、优质评测 | 可用，但需标注出处 |
| `还没核实` | AI 推断、间接引用、单一弱来源 | 可以提及，但必须标 `⏳ 还没核实` |
| `不能用` | 无法追溯、道听途说、明显冲突 | 不纳入正式依据 |

## 六、证据筛选

证据进入正文前必须过 4 个筛子：

1. **相关吗**：是不是直接服务当前题目
2. **可信吗**：来源级别够不够
3. **新吗**：有没有过时风险
4. **够吗**：是不是只是一面之词

任一不通过，不进入核心结论。

## 七、交叉验证

### 7.1 何时必须交叉验证

- 高影响判断
- 涉及价格、数据、规则、版本
- 单一来源容易带偏结论时

### 7.2 交叉验证标准

```
IF 关键结论只来自一个来源:
  优先寻找第二来源

IF 第二来源冲突:
  标注冲突，不强行统一

IF 找不到第二来源:
  标注"单一来源，可信度有限"
```

## 八、时效校验

| 类型 | 要求 | 处理方式 |
| --- | --- | --- |
| 价格 / 费用 | 高时效 | 标注日期，提醒以实际为准 |
| 行业趋势 / 市场数据 | 中时效 | 标注年份和时间窗口 |
| 软件 / 工具 / 版本 | 高时效 | 标注版本号和查询日期 |
| 你口述的业务信息 | 当次有效 | 跨对话时重新确认 |

## 九、引用与标注

1. 所有核心结论都要能回溯到来源
2. 不说"根据调研"，要说具体出处
3. 不能确认的内容统一标 `⏳ 还没核实`
4. 冲突证据不能静默吞掉，要显式说明

## 十、证据不足退出规则

以下情况不继续硬做：

- 关键维度根本没有可靠来源
- 证据之间强冲突，无法判断哪边更可信
- 现有证据不足以支撑核心结论

退出口径：

> 老板，这个点我目前没找到足够硬的证据。现在如果继续往下推，结论会虚。我建议要么补材料，要么把它先标成还没核实。

## 十一、边界提醒

- 本协议只管"找和验证"，不管"最后推荐谁"
- 本协议不决定检索目标，只执行已说清的检索目标
