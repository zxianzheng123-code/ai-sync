# 工具链协议

> 定义所有技能在执行过程中使用 SubAgent、进度文件、技能调度等基础设施时必须遵守的规范。核心问题：后台任务怎么派、进度怎么追踪、技能之间怎么衔接。各 SKILL.md 引用本协议。

## 一、SubAgent 分派契约

后台工作（不需要用户参与的）可以派 SubAgent 执行：

```
分派时必须传入：
  ① 任务描述：明确说清楚要干什么
  ② 必要输入：上游产物、参考文档、约束条件
  ③ 输出要求：产出什么格式、写到哪里
  ④ 边界约束：只读还是可写、不能碰什么

SubAgent 完成后，按以下标准格式回写到进度文件对应阶段的"执行结果"区域：
  · 执行状态：已完成 / 需返工
  · 产出文件：实际产出的文件路径列表
  · 执行摘要：1-3 句话概括做了什么、关键决策
  · 遗留问题：未解决的问题，无则写"无"
  · 完成日期：实际完成日期

总控读取结果，验收后向用户汇报

禁止：
  · SubAgent 直接面向用户对话
  · SubAgent 之间直接通信
  · SubAgent 修改未授权的文件
```

## 二、进度文件管理

长流程任务使用进度文件跟踪状态：

```
进度文件职责：
  · 记录当前走到哪个阶段
  · 每个阶段的任务说明、验收标准（由分配方编制）
  · 每个阶段的执行结果（由执行方回填）
  · 每个阶段的验收结论（由分配方回填）
  · 产出物清单和路径

进度文件位置：项目目录下 design/progress.md

进度文件标准格式：
  文件头（必填）：
    · 项目名称：项目/任务名
    · 创建日期：首次创建日期
    · 当前阶段：编号 + 阶段名（如 "②产品设计"）
    · 总控状态：规划中 / 执行中 / 已完成

  每个阶段必须包含：
    分配者填写 ──
    · 任务描述：本阶段做什么
    · 输入物：依赖的上游产物路径
    · 预期产出：本阶段应输出什么
    · 验收标准：逐条列出

    执行者填写 ──
    · 执行状态：未开始 / 进行中 / 已完成 / 需返工
    · 产出文件：实际产出的文件路径列表
    · 执行摘要：做了什么、关键决策
    · 遗留问题：未解决的问题，无则写"无"
    · 完成日期：实际完成日期

    分配者回填 ──
    · 验证结论：通过 / 需返工（附原因）

跨对话恢复：
  · 每次对话开头读取进度文件
  · 向用户汇报当前状态
  · 对时效敏感的信息重新确认
  · 用户确认后从暂停处继续
```

## 三、技能调度

造工具总控调度下游技能时的规范：

```
调度流程：
  1. 读取当前阶段对应的 SKILL.md
  2. 跟用户对话的步骤 → 总控直接按 SKILL.md 执行
  3. 后台工作步骤 → 派 SubAgent 执行
  4. 阶段完成 → 验收 → 汇报 → 用户确认 → 下一阶段

上下文传递：
  · 上游产出物路径写入进度文件
  · 下游技能启动时从进度文件读取上游产物
  · 不依赖对话记忆传递信息，一切以文件为准
```
