# 证据方法

> 事实核查维度的核心专业知识：如何验证证据的可靠性。

## 一、来源分级方法

| 级别 | 说明 | 处理方式 |
| --- | --- | --- |
| `硬证据` | 用户亲口说的、官方文档、一手数据 | 可直接作为核心依据 |
| `有参考价值` | 可追溯第三方资料、行业报告、优质评测 | 可用，但需标注出处 |
| `还没核实` | AI 推断、间接引用、单一弱来源 | 可以提及，但必须标 `⏳ 还没核实` |
| `不能用` | 无法追溯、道听途说、明显冲突 | 不纳入正式依据 |

## 二、证据筛选（4 个筛子）

1. **相关吗** — 是不是直接服务当前题目
2. **可信吗** — 来源级别够不够
3. **新吗** — 有没有过时风险
4. **够吗** — 是不是只是一面之词

任一不通过，不进入核心结论。

## 三、交叉验证标准

- 高影响判断、涉及价格/数据/规则/版本、单一来源容易带偏结论时必须交叉验证
- 关键结论只来自一个来源 → 优先寻找第二来源
- 第二来源冲突 → 标注冲突，不强行统一
- 找不到第二来源 → 标注"单一来源，可信度有限"

## 四、时效校验规则

| 类型 | 要求 | 处理方式 |
| --- | --- | --- |
| 价格 / 费用 | 高时效 | 标注日期，提醒以实际为准 |
| 行业趋势 / 市场数据 | 中时效 | 标注年份和时间窗口 |
| 软件 / 工具 / 版本 | 高时效 | 标注版本号和查询日期 |
| 用户口述的业务信息 | 当次有效 | 跨对话时重新确认 |

## 五、引用标注规范

1. 所有核心结论都要能回溯到来源
2. 不说"根据调研"，要说具体出处
3. 不能确认的内容统一标 `⏳ 还没核实`
4. 冲突证据不能静默吞掉，要显式说明

## 六、引用准确性核对

1. 引用内容是否与原文一致（逐字核对关键数字和结论）
2. 是否断章取义（检查上下文是否改变了原意）
3. 引用归属是否正确（张冠李戴检查）
4. 对比数据确认对比口径一致（同期、同范围、同定义）
