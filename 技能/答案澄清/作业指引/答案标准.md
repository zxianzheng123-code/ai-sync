# 答案标准

> 定义答案澄清工作流在产出「答案」时必须遵守的规则。本文件只管"解释、澄清、认知补全"这一类成果；不管多方案取舍，也不管把东西固化成方法或教材。

## 一、答案定义

「答案」指的是：

- 对一个已经说清的问题给出解释
- 对一个概念、现象、规则做澄清
- 给你一个可直接理解的认知补全

## 二、进入条件

进入「答案」前，至少满足：

1. 题目已经说清
2. 你不是在做多方案比较
3. 你不是在追求可重复执行的方法
4. 你不是要沉淀成教材或工具

不满足时的回退：

- 题目还没说清 → `问题拆解师`
- 需要比较 / 诊断 / 推荐 → `调研判断师`
- 需要 SOP / 规则 / 流程 → `方法提炼师`
- 需要课本 / 训练资料 / 体系文档 → `教材编写师`
- 需要系统 / 页面 / 应用 → `造工具总控`

## 三、答案结构

一个合格的答案至少包含：

1. **一句话答案** — 用最短的话直接回答问题
2. **简明解释** — 把一句话答案展开，让你理解为什么是这样
3. **边界或例外**（如有） — 什么情况下这个答案不成立、需要另说
4. **下一步建议**（如有） — 知道了这个答案之后，你可能需要做什么

## 四、证据使用边界

答案澄清可以做轻量事实核对，但不进入大规模信息搜集和多方案对比。

### 轻量核对 vs 系统检索的分界线

| 情况 | 属于 | 举例 |
|------|------|------|
| 来源数 ≤ 2，且不需要对比 | 轻量事实核对（答案澄清师可做） | "某公司成立年份"、"某产品的价格" |
| 来源数 > 2，或需要多方对比 | 系统检索（转调研判断师） | "三家供应商哪个更合适"、"这个方案有没有风险" |

**规则**：

- 可以引用少量硬证据来源
- 一旦发现需要系统检索、交叉验证、方案比较，立即上交给 `调研判断师`

## 五、边界提醒

- `答案` 不是 `判断` — 不做多方案取舍
- `答案` 不是 `方法` — 不产出可重复执行的 SOP
- `答案` 不是 `教材` — 不系统讲原理和案例
- `答案` 也不是 `工具需求` — 不产出系统或应用
